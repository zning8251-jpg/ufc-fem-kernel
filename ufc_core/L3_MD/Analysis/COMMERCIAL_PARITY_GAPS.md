# Analysis 域：商业参考求解器「全关键字 / 全步类型」对齐缺口登记

**目的**：把「暂时做不到的全量校准」拆成**可跟踪、可分批关闭**的缺口清单，避免与 [`CONTRACT.md`](CONTRACT.md) 中已声明的 **L3 职责边界**混淆。  
**范围**：以 **`L3_MD/Analysis`**（Step / Amplitude / Solver）为主视角；凡依赖 **KeyWord、L4_PH、L5_RT、L2** 的项一并列出归属，便于排期。  
**维护**：关闭或缩小缺口时，请同步更新本表状态列，并在 [`CONTRACT.md`](CONTRACT.md) 版本历史中留一行摘要。

**用语**：「商业参考」指常见隐式/显式有限元求解器的 **输入语言 + 步进/过程语义**（如 INP 风格 `*STEP` 内过程关键字族）；**不**在本文件复述第三方手册条文，避免版权与版本漂移问题；工程对齐以 **自选 golden 输入 + 可重复数值验收** 为准。

---

## 1. 为何无法「一次性校准完毕」

| 维度 | 说明 |
|------|------|
| **输入语言** | 全关键字 = 开放集合；参数组合与上下文合法性（步内允许哪些卡、互斥关系）随版本变化。 |
| **步类型** | `MD_Step_Proc.f90` 中 **`PROC_*` 与 UF 控制块**可表达较宽的过程枚举，但 **数值实现与 Runner 分发**在 **L5_RT / L2**；L3 仅有 Desc 与映射不等于行为一致。 |
| **验收** | 无 **封闭测试矩阵**（关键字子集 × 参考版本 × 容差）则「一致」不可判定。 |

## 3. 关键字侧（G-KW）——缺什么

以下为**能力类**缺口，不代表仓库零实现；具体已实现条目以 **`L3_MD/KeyWord`** 源码与 [`../KeyWord/CONTRACT.md`](../KeyWord/CONTRACT.md) 为准。

1. **覆盖率**：`MD_KW_Coverage` 类审计输出的是「本仓库注册/解析路径」对「自选关键字列表」的覆盖，**不是**对第三方手册全书的百分比。
2. **语义深度**：Lexer/Parser/AST 之后，**Mapper → 各域 Desc** 的字段级等价（默认值、单位、与步类型的合法组合）需 **逐关键字规格** + 测试。
3. **版本与方言**：同一关键字在不同参考版本间行为差异（参数重命名、废弃项）需 **显式版本矩阵**，否则「一致」无定义。
4. **错误行为**：非法组合、缺参、步外使用等，与参考软件的 **报错时机与文案级别** 对齐属于产品决策，通常弱于数值一致。

---

## 4. 步类型 / 过程侧（G-STEP-L3 / G-STEP-L5）——缺什么

1. **枚举与文档**：`MD_Step_Proc.f90` 中 **`PROC_*`、SSD 子变体、`ProcToRTStepType`** 等已较丰富；**缺口**是：与总合同 §六 **简化 `MD_STEP_*` 表** 的 **双向映射文档化**（避免读者以为 7 类即全集）。
2. **Runner 与物理**：各 `PROC_*` 是否在 **L5** 具备 **完整时间积分 / 特征值 / 耦合场** 实现，需按 PROC **逐条打勾**（当前未在本文件逐 PROC 维护，建议后续用表格或 issue 链）。
3. **步内关键字子树**：如 `*CONTROLS`、质量缩放、阻尼、重启动、自适应增量等，属于 **步控制 Desc**；与参考软件一致需 **INDATA 映射表 + L5 消费**，不单靠 L3。
4. **Step 合同引用的映射表**：[`Step/CONTRACT.md`](Step/CONTRACT.md) 引用 **INDATA_MAP**；若仓库内路径迁移，应固定真源链接并在此登记（避免「有引用无文件」）。

## 5. 幅值（G-AMP）——缺什么

以 [`Amplitude/CONTRACT.md`](Amplitude/CONTRACT.md) 为准，与「全类型一致」相关的主要缺口：

1. **USER / UAMP**：未注册子程序时的 **中性 `1.0_wp`** 与参考软件 **报错或别种默认** 可能不一致。
2. **PERIODIC 等**：UF 与域 **公式分支** 已文档化；与特定参考实现 **系数定义、相位约定** 的逐项对齐需 golden。
3. **谱/致动器/PSD 等**：当前 **L3 占位**；若要对齐参考软件，需 **扩展 Desc + 求值链 + L5 数据源**。

---

## 6. 求解器配置（G-SOLV）——缺什么

以 [`Solver/CONTRACT.md`](Solver/CONTRACT.md) 为准：

1. **桩类型落地**：`MD_LinearSolver_Desc` / `MD_NR_Algo` / `MD_Precond_Desc` 与 **L5 选路、L2 内核参数** 的一一绑定未在本仓库声明完成。
2. **运行时统计**：`MD_Solver_State` 等 **L3 占位**与 L5/L2 **真实迭代/残差** 同步策略未产品化。

---

## 7. 跨域与接线（G-XD）——缺什么

1. **重启 / 场输出 / 历史输出**：L3 Step 侧若仅 **Desc 或 stub 注释**，与参考 **`*RESTART`、`*OUTPUT` 族** 的端到端一致属 **Output + L5** 范围。
2. **接触与步**：摩擦、接触状态传递与 **显式/隐式步** 绑定，属 **Interaction + L5**。
3. **材料非线性 × 步**：NLGEOM、线性与非线性摄动等，属 **Material + Step + L5**。

---

## 8. 验收与「校准」可判定条件（G-VAL）

在缩小缺口前，建议先固定 **可判定** 目标，例如：

1. **关键字子集表**：列出首批支持的 `*...` 清单（含禁止组合说明）。
2. **步类型子集表**：从 `PROC_*` 中选子集，对应 **Runner 已实现** 列表。
3. **Golden 集**：每对 (关键字实例, 步类型) 至少 **一个** 可自动化对比的基准（解析结果或数值场在约定容差内）。
4. **参考版本列**：标明对照的参考软件 **主版本号**；跨版本不声称一致。
