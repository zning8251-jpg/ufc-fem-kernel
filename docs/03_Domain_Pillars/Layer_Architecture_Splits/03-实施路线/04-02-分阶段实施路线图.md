## 阶段 A：架构固化（4 周）

### A.1 依赖审计（1 周）

- **工具**：`python scripts/dep_audit.py`
- **输出**：`DEPENDENCY_AUDIT_REPORT.md`（含依赖图、循环依赖清单）
- **目标**：消除所有循环依赖，确保严格单向依赖

### A.2 Context 统一（1.5 周）

- **范围**：RT_RunJob、Implicit_RunStep、Explicit_RunStep
- **方法**：将线程局部状态改为显式 Context 参数传递
- **验证**：确保 Job 结束时 Context 完全清理

### A.3 模块边界文档（0.5 周）

- **输出**：每层 `README.md` 明确职责、禁止依赖、允许依赖
- **格式**：表格 + 依赖图 + 示例代码

### A.4 命名规范（1 周）

- **任务**：UF_/MD_/RT_/AP_ 前缀与层对应
- **输出**：`NAMING_MIGRATION_TABLE.md`（旧名→新名映射表）
- **执行**：分批迁移，每批完成后编译验证

## 阶段 B：核心域收敛（6 周）

### B.1 StepDriver 聚合（1.5 周）

- **目标**：12→2 核心模块（Core + Strategies）
- **策略**：按职责分离核心逻辑与策略选择
- **检查点**：每个子程序 ≤ 4000 行

### B.2 Material 完善（2.5 周）

- **目标**：MatLib 子域聚合完成，UMAT 调度统一
- **策略**：遵循"模块合并与子程序拆分"原则
- **检查点**：文件大小 ≤ 3000 行，子程序 ≤ 4000 行

### B.3 Geometry 聚合（1.5 周）

- **目标**：15→5 核心模块
- **策略**：按几何类型聚合（Part/Assembly/Instance/Feature/Operation）
- **检查点**：每个子程序 ≤ 4000 行

### B.4 Contact 增强（0.5 周）

- **目标**：摩擦、自适应、通用接触策略
- **策略**：在现有聚合基础上扩展功能

## 阶段 C：统一场论落地（6 周）

### C.1 UF_UnifiedField_Core（2 周）

- **内容**：场类型定义、场状态管理、场管理器
- **接口**：统一场方程 `C·φ̇ + K·φ + f_int = f_ext`
- **检查点**：每个子程序 ≤ 4000 行

### C.2 UF_UnifiedField_TimeIntegration（1.5 周）

- **内容**：θ/Newmark/广义α 统一时间积分框架
- **策略**：策略模式实现不同积分方法
- **检查点**：每个积分算法子程序 ≤ 2000 行

### C.3 UF_MultiPhysics_Coupling（1.5 周）

- **内容**：单块/交替/分区迭代三种耦合策略
- **接口**：统一耦合接口，策略可插拔
- **检查点**：每个耦合策略子程序 ≤ 3000 行

### C.4 物理模型整合（1 周）

- **内容**：结构/热/孔隙/THM 统一接口
- **方法**：适配器模式统一现有物理模型
- **验证**：热-力耦合算例可运行

## 阶段 D：求解与性能（6 周）

### D.1 线性求解增强（1.5 周）

- **自动选择**：根据矩阵特性选择直接/迭代求解器
- **预条件**：ILU、AMG、Jacobi 等预条件器
- **GPU 迭代**：CG/GMRES 等迭代方法 GPU 化
- **检查点**：每个求解器子程序 ≤ 3000 行

### D.2 非线性诊断（1.5 周）

- **收敛分析**：残差历史、收敛率、诊断报告
- **自适应策略**：自动调整步长、阻尼、线搜索参数
- **检查点**：诊断子程序 ≤ 2000 行，策略子程序 ≤ 3000 行

### D.3 MPI 分布式（2 周）

- **数据分解**：域分解、图划分、负载均衡
- **通信**：点对点、集合通信、异步通信
- **检查点**：通信子程序 ≤ 2000 行，分解子程序 ≤ 3000 行

### D.4 GPU 全链路（1 周）

- **装配 GPU 化**：单元刚度矩阵组装 GPU 内核
- **求解 GPU 化**：稀疏矩阵-向量乘、迭代求解 GPU 化
- **检查点**：每个 GPU 内核 ≤ 1000 行（.cu 文件）

## 阶段 E：质量与文档（4 周）

### E.1 错误处理体系（1 周）

- **统一错误码**：IF_Err_Type 定义所有错误码
- **错误恢复**：可恢复错误自动重试机制
- **日志链路**：统一日志格式，支持多级别输出
- **检查点**：错误处理子程序 ≤ 1500 行

### E.2 测试体系（1.5 周）

- **单元测试**：每个核心子程序至少 1 个测试用例
- **集成测试**：模块间接口测试
- **回归测试**：历史算例回归验证
- **压力测试**：大规模、长时间运行测试
- **目标**：覆盖率 ≥ 80%

### E.3 文档体系（1 周）

- **架构文档**：六层架构详细说明、依赖图
- **API 文档**：所有公共接口文档（Doxygen 格式）
- **用户文档**：使用指南、示例、常见问题
- **开发者文档**：贡献指南、代码规范、扩展指南

### E.4 监控与诊断（0.5 周）

- **性能监控**：关键路径计时、热点分析
- **内存监控**：内存使用统计、泄漏检测
- **收敛监控**：迭代历史、收敛诊断报告
