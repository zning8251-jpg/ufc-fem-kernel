## 概述

L3_MD层的关键字解析系统负责解析ABAQUS兼容的INP文件，将关键字转换为UFC内部数据结构，构建完整的模型树。这是前处理的核心功能，确保UFC能够读取和理解ABAQUS格式的输入文件。

**核心功能**：

- INP文件解析
- 关键字识别和分类
- 关键字到数据结构的映射
- 模型树构建
- 关键字验证和错误处理

## 关键字分类体系

### 模型定义关键词（L3_MD层）

**空间建模**：

- `*NODE`：节点定义
- `*ELEMENT`：单元定义
- `*SURFACE`：表面定义
- `*RIGID BODY`：刚体定义
- `*ASSEMBLY`：装配定义
- `*PART`：部件定义
- `*INSTANCE`：实例定义

**材料定义**：

- `*MATERIAL`：材料定义
- `*ELASTIC`：弹性材料
- `*PLASTIC`：塑性材料
- `*HYPERELASTIC`：超弹性材料
- `*CREEP`：蠕变材料
- `*DAMAGE`：损伤材料
- `*DENSITY`：密度定义

**截面定义**：

- `*SOLID SECTION`：实体截面
- `*SHELL SECTION`：壳截面
- `*BEAM SECTION`：梁截面
- `*MEMBRANE SECTION`：膜截面

### 载荷与边界条件关键词（L3_MD层）

**边界条件**：

- `*BOUNDARY`：边界条件定义
- `*INITIAL CONDITIONS`：初始条件定义
- `*PREDEFINED FIELD`：预定义场

**载荷**：

- `*CLOAD`：集中载荷
- `*DLOAD`：分布载荷
- `*DSLOAD`：分布表面载荷
- `*DFLUX`：分布热流
- `*CFLUX`：集中热流

### 约束关键词（L3_MD层）

- `*TIE`：绑定约束
- `*COUPLING`：耦合约束
- `*MPC`：多点约束
- `*EQUATION`：方程约束

### 接触关键词（L3_MD层）

- `*CONTACT PAIR`：接触对定义
- `*SURFACE INTERACTION`：表面相互作用
- `*CONTACT`：接触定义
- `*FRICTION`：摩擦定义

### 分析步关键词（L5_RT层）

**分析类型**：

- `*STEP`：分析步定义
- `*STATIC`：静力分析
- `*DYNAMIC`：动力分析
- `*FREQUENCY`：频率提取
- `*BUCKLE`：屈曲分析
- `*HEAT TRANSFER`：热传导分析
- `*COUPLED TEMPERATURE-DISPLACEMENT`：热力耦合分析

**求解控制**：

- `*CONTROLS`：求解控制参数
- `*RESTART`：重启动设置

### 输出关键词（L5_RT层）

- `*OUTPUT`：输出定义
- `*FIELD OUTPUT`：场输出
- `*HISTORY OUTPUT`：历史输出
- `*NODE OUTPUT`：节点输出
- `*ELEMENT OUTPUT`：单元输出

## 关键字层次结构

ABAQUS关键字采用层次化结构：

**Level 0（顶层关键字）**：
- `*HEADING`
- `*PREPRINT`
- `*PART`
- `*ASSEMBLY`
- `*STEP`

**Level 1（一级子关键字）**：
- `*NODE`（在`*PART`或`*ASSEMBLY`内）
- `*ELEMENT`（在`*PART`或`*ASSEMBLY`内）
- `*MATERIAL`（在`*PART`或`*ASSEMBLY`内）
- `*STATIC`（在`*STEP`内）
- `*BOUNDARY`（在`*STEP`内）

**Level 2（二级子关键字）**：
- `*ELASTIC`（在`*MATERIAL`内）
- `*PLASTIC`（在`*MATERIAL`内）
- `*FIELD OUTPUT`（在`*STEP`内）

## 关键字解析器架构

### 核心解析器模块

```fortran
! L3_MD/Keyword/MD_KW_Parser.f90
MODULE MD_KW_Parser
    USE IF_Prec
    USE MD_KW_Types
    USE MD_KW_Categories
    IMPLICIT NONE
    
    ! 关键词解析接口
    INTERFACE Parse_Keyword
        MODULE PROCEDURE Parse_Node_Keyword
        MODULE PROCEDURE Parse_Element_Keyword
        MODULE PROCEDURE Parse_Material_Keyword
        MODULE PROCEDURE Parse_Load_Keyword
        MODULE PROCEDURE Parse_Boundary_Keyword
    END INTERFACE Parse_Keyword
    
    ! 关键词注册表
    TYPE, PUBLIC :: KeywordRegistry
        CHARACTER(LEN=KW_MAX_NAME_LEN) :: keyword_name
        INTEGER(i4) :: keyword_category
        INTEGER(i4) :: keyword_id
        INTEGER(i4) :: keyword_level          ! 关键字层次（0/1/2）
        CHARACTER(LEN=KW_MAX_NAME_LEN) :: parent_keyword  ! 父关键字
        PROCEDURE(ParseKeywordProc), POINTER :: parse_proc
    END TYPE KeywordRegistry
    
    ! 解析INP文件
    SUBROUTINE MD_ParseINPFile(filename, model, status)
        CHARACTER(*), INTENT(IN) :: filename
        TYPE(MD_ModelDesc), INTENT(OUT) :: model
        INTEGER(i4), INTENT(OUT) :: status
    END SUBROUTINE
    
    ! 解析单个关键字
    SUBROUTINE MD_ParseKeyword(keyword_line, model, status)
        CHARACTER(*), INTENT(IN) :: keyword_line
        TYPE(MD_ModelDesc), INTENT(INOUT) :: model
        INTEGER(i4), INTENT(OUT) :: status
    END SUBROUTINE
END MODULE MD_KW_Parser
```

## INP文件解析流程

### 解析流程

```
1. 打开INP文件
2. 逐行读取
3. 识别关键字行（以*开头）
4. 查找关键字注册表
5. 调用对应的解析函数
6. 构建模型树节点
7. 读取数据行（直到下一个关键字或END关键字）
8. 验证关键字参数
9. 更新模型树
10. 重复步骤2-9直到文件结束
```

### 关键字行格式

**基本格式**：

```
*KEYWORD, PARAM1=value1, PARAM2=value2
```

**示例**：

```
*MATERIAL, NAME=STEEL
*ELASTIC
200000.0, 0.3
*PLASTIC
300.0, 0.0
400.0, 0.1
```

### 数据行格式

**节点数据**：

```
*NODE
1, 0.0, 0.0, 0.0
2, 1.0, 0.0, 0.0
3, 1.0, 1.0, 0.0
```

**单元数据**：

```
*ELEMENT, TYPE=C3D8
1, 1, 2, 3, 4, 5, 6, 7, 8
2, 5, 6, 7, 8, 9, 10, 11, 12
```

### 映射示例

**节点关键字解析**：

```fortran
SUBROUTINE Parse_Node_Keyword(keyword_line, data_lines, model, status)
    CHARACTER(*), INTENT(IN) :: keyword_line
    CHARACTER(*), INTENT(IN) :: data_lines(:)
    TYPE(MD_ModelDesc), INTENT(INOUT) :: model
    INTEGER(i4), INTENT(OUT) :: status
    
    INTEGER(i4) :: i, node_id
    REAL(wp) :: coords(3)
    TYPE(MD_NodeDesc) :: node
    
    DO i = 1, SIZE(data_lines)
        ! 解析节点数据：node_id, x, y, z
        READ(data_lines(i), *) node_id, coords(1), coords(2), coords(3)
        
        ! 创建节点描述
        node%node_id = node_id
        node%coordinates = coords
        
        ! 添加到模型树
        CALL MD_AddNode(model, node, status)
    END DO
END SUBROUTINE
```

**材料关键字解析**：

```fortran
SUBROUTINE Parse_Material_Keyword(keyword_line, model, status)
    CHARACTER(*), INTENT(IN) :: keyword_line
    TYPE(MD_ModelDesc), INTENT(INOUT) :: model
    INTEGER(i4), INTENT(OUT) :: status
    
    CHARACTER(LEN=64) :: material_name
    TYPE(MD_MaterialDesc) :: material
    
    ! 解析关键字参数：*MATERIAL, NAME=STEEL
    CALL ParseKeywordParams(keyword_line, 'NAME', material_name, status)
    
    ! 创建材料描述
    material%material_name = material_name
    
    ! 添加到模型树
    CALL MD_AddMaterial(model, material, status)
END SUBROUTINE
```

## 模型树构建

### 模型树结构

```
MD_ModelDesc
├── heading: CHARACTER(256)
├── parts: MD_PartDesc(:)
│   ├── part_name: CHARACTER(64)
│   ├── nodes: MD_NodeDesc(:)
│   ├── elements: MD_ElemDesc(:)
│   └── materials: MD_MaterialDesc(:)
├── assembly: MD_AssemblyDesc
│   ├── instances: MD_InstanceDesc(:)
│   └── ...
└── steps: RT_StepDesc(:)
    ├── step_name: CHARACTER(64)
    ├── analysis_type: INTEGER
    └── ...
```

### 模型树构建流程

```
1. 创建根节点（MD_ModelDesc）
2. 解析*PART关键字 → 创建Part节点
3. 解析*NODE关键字 → 添加到当前Part
4. 解析*ELEMENT关键字 → 添加到当前Part
5. 解析*MATERIAL关键字 → 添加到当前Part
6. 解析*END PART → 完成Part构建
7. 解析*ASSEMBLY关键字 → 创建Assembly节点
8. 解析*INSTANCE关键字 → 创建Instance节点
9. 解析*STEP关键字 → 创建Step节点
10. 解析*END STEP → 完成Step构建
```

## 关键字验证

### 验证规则

1. **必需参数检查**：检查关键字是否包含必需的参数
2. **参数类型检查**：检查参数类型是否正确（整数、实数、字符串）
3. **参数范围检查**：检查参数是否在有效范围内
4. **层次关系检查**：检查关键字是否在正确的父关键字内
5. **依赖关系检查**：检查关键字依赖的其他关键字是否存在

### 验证示例

```fortran
SUBROUTINE Validate_Material_Keyword(material, model, status)
    TYPE(MD_MaterialDesc), INTENT(IN) :: material
    TYPE(MD_ModelDesc), INTENT(IN) :: model
    INTEGER(i4), INTENT(OUT) :: status
    
    ! 1. 检查材料名称是否唯一
    IF (MD_MaterialExists(model, material%material_name)) THEN
        status = STATUS_DUPLICATE_MATERIAL
        RETURN
    END IF
    
    ! 2. 检查是否至少有一个材料行为（如*ELASTIC）
    IF (.NOT. ASSOCIATED(material%elastic_desc) .AND. &
        .NOT. ASSOCIATED(material%plastic_desc)) THEN
        status = STATUS_MISSING_MATERIAL_BEHAVIOR
        RETURN
    END IF
    
    status = IF_STATUS_OK
END SUBROUTINE
```

## 使用示例

```fortran
USE MD_KW_Parser
USE MD_Model_Def

TYPE(MD_ModelDesc) :: model
INTEGER(i4) :: status

! 解析INP文件
CALL MD_ParseINPFile('model.inp', model, status)

IF (status == IF_STATUS_OK) THEN
    WRITE(*,*) '解析成功'
    WRITE(*,*) '部件数：', SIZE(model%parts)
    WRITE(*,*) '分析步数：', SIZE(model%steps)
ELSE
    WRITE(*,*) '解析失败，错误码：', status
END IF
```
