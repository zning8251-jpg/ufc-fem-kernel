## 治理目标

### 目标1：统一命名规范
**所有材料族文件统一使用：`MD_Mat_族名_模型名_功能后缀.f90`**

**族名缩写标准：**
```
Elas      - 弹性材料族
Plast     - 塑性材料族
Hyper     - 超弹性材料族
Damage    - 损伤材料族
Creep     - 蠕变材料族
Acou      - 声学材料族
Therm     - 热材料族
Visco     - 粘弹性材料族
Comp      - 复合材料族
Geo       - 岩土材料族
User      - 用户自定义材料族
```

**功能后缀标准：**
```
_Def      - 族级定义文件（TYPE定义、常量定义）
_Core     - 族级核心文件（Validate、Populate等）
_Brg      - 族级桥接文件（L4路由）
_Compat   - 族级兼容文件（向后兼容）
无后缀    - 具体材料模型文件
```

### 目标2：明确子域职责

**材料族子域（11个）：**
- Elas、Plast、Hyper、Damage、Creep、Acoustic、Thermal、Viscoelas、Composite、Geo、User
- 每个族包含：族级文件（Def、Core、Brg、Compat）+ 具体材料模型文件

**基础设施子域（3个）：**
- **Contract**：定义材料域的接口合约（MD_Mat_Def.f90、MD_Mat_Ids.f90等）
- **Dispatch**：材料模型的分发调度（MD_Mat_Lib.f90等）
- **Registry**：材料模型的注册表（MD_MatReg_Ops.f90等）
