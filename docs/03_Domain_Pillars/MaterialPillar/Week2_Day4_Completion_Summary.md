## 架构分析

### 当前架构（已经是统一的）

```
Element计算
  ↓
RT_Mat_Core::RT_Mat_Dispatch_Stress
  ↓
RT_Mat_Dispatch_Table（统一调度表）
  ↓ (通过mat_type路由)
L4层计算函数（PH_Mat_Update_Stress）
```

**关键特性：**
- ✅ 单一入口点（RT_Mat_Core）
- ✅ 统一调度表（所有材料族共享）
- ✅ O(1)查找性能（通过mat_type）
- ✅ 清晰的L5→L4委托

### RT_Mat_XXX_Core的作用

**重新理解：**
- RT_Mat_Elas_Core、RT_Mat_Plast_Core等模块
- 不是"重复的调度机制"
- 而是"材料族特定的调度表管理"
- 作为RT_Mat_Core的补充，不是重复

**分层设计：**
```
RT_Mat_Core (统一调度入口)
  ↓
RT_Mat_XXX_Core (材料族特定的调度表)
  ↓
L4层计算函数
```
