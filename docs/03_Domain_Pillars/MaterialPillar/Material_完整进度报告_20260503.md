## 二、架构设计特点

### 2.1 严格三层嵌套

```fortran
TYPE :: MD_Mat_XXX_Desc
  INTEGER(i4) :: family_type      ! Level 1: 主族（11个）
  INTEGER(i4) :: sub_type         ! Level 2: 变体（65个）
  INTEGER(i4) :: property_flags   ! Level 3: 属性（7个位标志）
END TYPE
```

**三层嵌套枚举**：
- Level 1：11个主族（ELASTIC/PLASTIC/HYPERELASTIC/DAMAGE/CREEP/VISCOELASTIC/GEOTECHNICAL/COMPOSITE/THERMAL/ACOUSTIC/USER）
- Level 2：65个辅TYPE（每个主族3-15个变体）
- Level 3：7个属性标志（TEMP_DEP/FIELD_DEP/RATE_DEP/ANISO/COUPLED/NONLOCAL/USER）

### 2.2 四类TYPE完整

**Desc TYPE**：静态描述符（L3_MD）
- 材料参数
- 派生参数
- 初始化标志

**State TYPE**：运行时状态（L4_PH/L5_RT）
- 应力/应变
- 内部变量（塑性应变、背应力、损伤变量等）
- 历史变量

**Algo TYPE**：算法控制（L4_PH）
- 积分方法
- 迭代参数
- 切线类型

**Ctx TYPE**：迭代级工作区（L4_PH）
- 临时变量
- 工作数组
- 中间结果

### 2.3 L3/L4/L5三层贯通

```
用户输入（ABAQUS inp）
    ↓
L3_MD: 模型定义层
    ├─ 解析关键字
    ├─ 创建Desc
    ├─ 验证参数
    └─ 注册材料
    ↓ Populate (Brg)
L4_PH: 物理计算层
    ├─ 构建刚度矩阵
    ├─ 计算应力
    ├─ 计算切线
    └─ 更新状态
    ↓ Dispatch (RT)
L5_RT: 运行时调度层
    ├─ 路由调度
    ├─ State管理
    └─ Commit/Rollback
```

### 2.4 命名规范统一

完全符合UFC_命名规范_v3.0：

| 场景 | 公式 | 示例 |
|------|------|------|
| TYPE定义 | `{层}_{域}_{功能}_{四型}` | `MD_Mat_Elas_Desc` |
| MODULE/文件 | `{层}_{域}_{功能}[_{角色}].f90` | `MD_Mat_Elas_Core.f90` |
| 过程名 | `{层}_{域}[_{功能}]_{Verb}[_{Object}]` | `MD_Mat_Elas_Create_Isotropic` |

**角色后缀**：
- `_Def` - TYPE定义
- `_Core` - 核心实现
- `_Brg` - 桥接
- `_Eval` - 求值入口
- `_Compat` - 兼容层
