# 有限元内核验证技能

本技能用于将**验证体系**（本构、数值、物理一致性及标准测试问题）落实为可执行清单与回归策略，支撑验证驱动开发与持续集成。

---

## 何时使用本技能

- 用户提到 **验证**、**回归**、**验收**、**验证驱动开发**
- 需要设计或检查 **Patch Test**、**Cook's Membrane**、**Cantilever**、**NAFEMS**、**ABAQUS 验证手册** 对照
- 为**新算法、新本构、新单元族**设计验证用例或回归用例
- 讨论 **CI 中自动运行** 的验证套件、基准结果与容差

## 验证维度速查（与 02-07 对齐）

1. **本构关系**：客观性、热力学一致性、材料对称性；实现参考 `L4_PH/Material/PH_Mat_Valid.f90`。
2. **数值方法**：Patch Test、收敛率、时间积分稳定性（A/L 稳定性）。
3. **物理一致性**：能量守恒、动量守恒（保守系统/无外力系统）。
4. **标准测试问题**：Cook's Membrane、Cantilever、Patch Test、弹性波；与 NAFEMS/ABAQUS 验证手册对照。
5. **实现位置**：`L4_PH/Valid/`、`L2_NM/Valid/` 等。

---

## 使用本技能时的产出建议

- **标准题清单**：每类算法/本构/单元族对应的标准题、参考解或容差、输入/输出约定。
- **回归用例设计**：哪些用例进入 CI、触发条件（提交到某域/某层）、基准结果存储方式。
- **与 02-07 的映射**：新增验证项时，在 02-07 或配套清单中补充条目，保持可追溯。

---

## 与 fem-kernel-architecture 的衔接

- 架构技能中的「测试与验收」小节与本文档一致；域级或四链迭代完成后，用本技能检查验证覆盖是否跟上。
- 新增域/新算法时，优先在本文档（或项目内验证清单）中增加对应验证项与用例，再实现代码，形成验证驱动闭环。
