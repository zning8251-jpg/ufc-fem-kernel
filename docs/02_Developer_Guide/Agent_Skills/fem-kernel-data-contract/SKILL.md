# 有限元内核数据契约技能

本技能用于在六层架构下，将**数据链**显式化为可维护的**契约**：层间/域间输入输出类型、稳定字段、版本与兼容策略、以及可选的序列化约定，确保演进时调用方不意外破坏。

---

## 何时使用本技能

- 用户提到 **数据契约**、**接口契约**、**层间数据**、**数据链**
- 讨论 **版本兼容**、**向后兼容**、**演进不破坏**
- 需要 **序列化**、**反序列化**、**持久化**、**检查点/重启** 的数据格式约定
- 设计或评审**跨层/跨域**传递的 Desc、Ctx、State 类型与字段

## R 与 θ 在四型中的落位（缩略）

与 **AI-ready 层 2/3**、**非线性/步进求解** 相关的契约要点（细节见上表契约卡）：

- **全局 R（n_dof）**：**L5_RT** 用步进统一 `Arg` 内指针（如 `RT_SD_Arg%f_global`）；**L2_NM** 用**形参**或回调 `R(:)`。**不要**把整条 R 塞进 **`NM_Solver` 的 `SolverStats`**（该型仅范数/迭代等标量）或 **`RT_AI_*_Ctx`** 作主存。
- **单元残差块**：**L4_PH** → **`State%rhs`**，由 **`Algo%compute_rhs`** 控制。
- **θ、伴随模式、设计变量索引**：经 **Desc（维数/布局）+ Ctx（当前值/活动集）** 传入；**禁止** L4 内 **SAVE** 或未合同化的模块可变状态。

## 契约维度

| 维度 | 内容 |
|------|------|
| **类型边界** | 某层/某域对外暴露的类型（如 `MD_Step_Desc`、`RT_Step_Ctx`）；仅通过类型名与公开成员传递，无裸数组/隐式全局 |
| **稳定字段** | 哪些字段为「稳定 API」、哪些为「内部保留可变更」；新增字段的默认值或兼容读旧版本 |
| **版本与兼容** | 类型或模块版本号（若需要）；旧数据/旧 INP 的读取策略；废弃字段的过渡期与迁移说明 |
| **序列化（可选）** | 检查点、重启、调试导出时的格式（如二进制布局、或与 INP/结果格式的对应）；反序列化时的校验与默认值 |

## 数据链契约检查清单

- [ ] 跨层/跨域调用是否**仅通过已声明的类型**（Desc/Ctx/State）传递？是否存在直接访问对方层内部类型或成员？
- [ ] 每个对外类型是否有**文档化字段表**（名称、含义、合法范围、默认值）？
- [ ] 若类型会持久化或跨版本：是否有**版本标识**与**兼容读**策略？
- [ ] Bridge 的输入/输出类型是否在 02-04 或等价文档中**显式列出**？
- [ ] 数据链文档是否与**四链-数据链**描述一致，便于变更时做影响分析？
- [ ] 若涉及灵敏度/优化：**全局 R** 是否未误写入仅含标量的 **Solver State** 或窄 **Ctx**？**θ** 是否经 **Desc/Ctx** 合同注入（参见 `L3_MD/contracts/CONTRACT_R_Theta_FourKind.md`）？

## 产出建议

- **契约表**：按层/域或按 Bridge 列出「输入类型 → 输出类型」或「调用方 → 被调用方类型」；标注稳定/可扩展字段。
- **版本与兼容策略**：新增字段的规则、废弃字段的流程、旧数据读取的测试要求。
- **与 02-05 的同步**：数据流文档中涉及的类型与本节契约表一致，并在变更时同步更新。

## 与 fem-kernel-architecture 的衔接

- 架构技能中的「数据链视角」关注数据从 L3→L4→L5→L2 的路径与可追溯性；本技能将这条路径**固化为契约**（类型、字段、版本），使「数据链可追溯」可落地、可审查。
- 域级迭代的「阶段 2：设计决策」中，新增或修改跨层类型时，应在本契约下补充或更新条目，并在「阶段 4：文档沉淀」中更新契约表与数据流文档。
