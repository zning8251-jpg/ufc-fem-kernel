# UFC @templates 四大类角色约定清单

**版本**: v1.0  
**签发日期**: 2026-03-30  
**适用范围**: `UFC/docs/templates/` 目录下所有 `*_Types.f90` 模板文件  
**配套技能**: `ufc-structured-io` (约束 _Proc 接口)  

---

## 一、四大类后缀总表

| 后缀 | 全称 | 职责 | 路径属性 | 访问频率 | 典型字段 |
|------|------|------|----------|----------|----------|
| **Desc** | Descriptor | 模型元信息、静态配置、材料参数 | 冷路径 | 一次写入，多次读取 | `mat_id`, `model_name`, `nprops`, `props(:)` |
| **State** | State | 计算过程动态状态、历史变量、响应输出 | 热路径 | 高频读写 | `stress(6)`, `statev(:)`, `converged`, `iterations` |
| **Algo** | Algorithm | 求解器参数、迭代控制、超参数 | 冷路径 | 迭代内只读 | `max_iter`, `tolerance`, `integ_scheme` |
| **Ctx** | Context | 跨层/跨域传递的增量驱动输入、临时缓冲 | 热路径 | 每增量填充 | `dstran(6)`, `dfgrd1(3,3)`, `time`, `coords(3)` |

**设计原则**: 
- Desc/State 通常归属 MD_层（模型描述）
- Ctx/Algo 通常归属 PH_层（物理计算）
- RT_层仅携带 Ctx（运行时上下文，无 Desc/State/Algo）

## 三、跨域聚合模式

### 3.1 Domain_Ctx 聚合规范 (`RT_Domain_Types.f90`)

所有 L5_RT 域通过 `RT_XXX_Domain_Ctx` 聚合下层类型：

```fortran
TYPE :: RT_Mat_Domain_Ctx
  TYPE(RT_Com_Base_Ctx), POINTER :: com_ctx => NULL()  ! Level 2
  TYPE(PH_Mat_Base_Ctx)          :: ph_ctx             ! L4_PH
  TYPE(PH_Mat_Base_State), POINTER :: ph_state => NULL()
  REAL(wp)                       :: pnewdt = 1.0_wp    ! 反馈标量
END TYPE
```

**聚合规则**:
1. 必须含 `com_ctx` 指针（Level 2 框架通道）
2. 嵌入 `PH_XXX_Base_Ctx` 值类型（非指针）
3. 使用 `PH_XXX_Base_State` 指针（指向 owned state）
4. 含 `pnewdt` 标量（热路径反馈）

## 四、命名合规性检查清单

### 4.1 TYPE 命名规则

✅ **合规**:
- `MD_Mat_Base_Desc` — 层前缀 + 域名 + 角色后缀
- `PH_Elem_VUEL_Ctx` — 子程序专属类型
- `RT_Asm_Init_In` — _Proc 操作输入
- `RT_Asm_Init_Out` — _Proc 操作输出

❌ **违规**:
- `MaterialDesc` — 缺少层前缀
- `MD_Material_Description` — 未用标准后缀
- `RT_Assembly_Input` — 应用 `_In` 而非 `Input`

### 4.2 _In/_Out 配对规则

每个操作必须有成对的 `_In` / `_Out` TYPE：

```fortran
TYPE :: RT_Asm_Init_In
  INTEGER(i4) :: mode = 0_i4
END TYPE

TYPE :: RT_Asm_Init_Out
  LOGICAL :: initialized = .FALSE.
  TYPE(ErrorStatusType) :: status  ! 必须
END TYPE
```

**禁令**:
- ❌ `_In` 含 `ALLOCATABLE` 字段
- ❌ `_In` 内嵌四大类（Desc/State/Algo/Ctx）
- ❌ `_Out` 缺少 `ErrorStatusType`
- ❌ TYPE 体内使用 `INTENT(...)` 属性

## 五、模板版本校验

### 5.1 有效性检查

使用前必须确认模板头部注释含版本号：

```fortran
! Module: MD_Mat_Types                                           [Template v3.1]
! Module: PH_Elem_Types                                          [Template v4.0]
! Module: RT_Assembly_Types                                      [Template v1.0]
```

**版本演进**:
- v3.x → v4.0: 移除冗余 Algo（如 `PH_Elem_Base_Algo`）
- v3.1: 修复漏洞（如 `MD_Mat_Base_State` 增加 `stran(6)`）
- v3.2: 增加子程序专属 Desc（如 `MD_Mat_CREEP_Desc`）

### 5.2 校验工具

执行以下命令验证模板完整性：

```bash
# 检查所有模板是否含版本注释
grep -l "Template v" UFC/docs/templates/*_Types.f90 | wc -l

# 检查是否存在非法 INTENT（应在 TYPE 外）
grep -n "INTENT(" UFC/docs/templates/*_Types.f90 | grep -v "SUBROUTINE\|FUNCTION"

# 检查四大类后缀覆盖率
grep -o "TYPE.*::.*_[DSAC][a-z]*" UFC/docs/templates/*.f90 | sort | uniq -c
```

## 六、与 ufc-structured-io 的映射关系

| 本文档约束对象 | ufc-structured-io 约束对象 | 协同规则 |
|---------------|---------------------------|----------|
| TYPE 定义结构 | _Proc 接口签名 | _Proc 的 `desc` 参数必须是 `TYPE(RT_XXX_Base_Desc)` |
| 四大类字段成员 | _In/_Out TYPE 内容 | _In 不得内嵌四大类，必须拆分为独立参数 |
| Ctx 零分配约束 | 热路径 INTENT 禁令 | Ctx 字段不得有 `ALLOCATABLE`，INTENT 仅用于接口 |
| State 高频写特性 | _Out 必须含 `ErrorStatusType` | State 更新与错误传播同步 |

**联合检查流程**:
1. 先用本文档验证 TYPE 定义符合四大类角色
2. 再用 `ufc-structured-io` 验证 _Proc 接口符合六参数签名
3. 最后交叉验证：_Proc 使用的 TYPE 必须在 templates 中有定义

## 八、常见违规模式速查

| 违规编号 | 描述 | 症状 | 修复动作 |
|---------|------|------|---------|
| **V-01** | MD 层携带 Ctx | `MD_XXX_Ctx` 出现在 MD_文件 | 迁移至 PH_层或删除 |
| **V-02** | PH 层携带 Desc | `PH_XXX_Desc` 出现在 PH_文件 | 迁移至 MD_层或删除 |
| **V-03** | RT 层携带 State | `RT_XXX_State` 独立存在 | 嵌入 Domain_Ctx 或改用指针 |
| **V-04** | Ctx 含 ALLOCATABLE | `REAL(wp), ALLOCATABLE :: arr(:)` in Ctx | 改为固定大小数组或指针 |
| **V-05** | State 缺 converged | 无 `LOGICAL :: converged` 字段 | 添加收敛标记 |
| **V-06** | Algo 缺 tolerance | 无 `REAL(wp) :: tolerance` 字段 | 添加容差参数 |
| **V-07** | Desc 缺 is_initialized | 无 `LOGICAL :: is_initialized` | 添加初始化标记 |
| **V-08** | 命名不含层前缀 | `MaterialDesc` 而非 `MD_Mat_Base_Desc` | 重命名为标准格式 |
| **V-09** | 缺少 ErrorStatusType | `_Out` TYPE 无 `status` 字段 | 添加错误状态 |
| **V-10** | TYPE 体内用 INTENT | `REAL(wp), INTENT(IN) :: val` in TYPE | 删除 INTENT |
| **V-11** | Desc TBP 冗余转发 | 模块内 `Subroutine *_TBP` 仅 `CALL MD_Mat_*_Foo(...)` | 删除 `*_TBP`；TBP 直接 `=> MD_Mat_*_Foo`，`Foo` 首参 `CLASS(Desc)` |
