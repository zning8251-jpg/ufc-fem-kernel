# 63 域/子域 × §0～§11 补全矩阵模板

本文用于追踪 UFC 所有域/子域在总纲 §0～§11 维度上的补全情况，作为实际执行与验收清单。

## 1. 使用说明

- 每一行对应一个域或子域。
- 每一列对应总纲中的一个章节。
- 状态建议：
  - `✅`：已完成
  - `🟡`：部分完成
  - `❌`：未完成
  - `NA`：不适用
- “待补文档区域”用于指明缺失内容应该补到哪里。
- “备注”用于记录依赖、迁移、白名单或其他说明。

## 4. 填写规则

1. **一域一行**
   - 每个域或子域单独占一行，不要合并不同职责对象。

2. **按章节逐项检查**
   - 若该章节已有正式文档，填 `✅`。
   - 若仅有草稿或部分材料，填 `🟡`。
   - 若没有对应内容，填 `❌`。
   - 若该章节不适用于该域，填 `NA`。

3. **补全时必须写明去向**
   - “待补文档区域”必须指向具体目录或文件；
   - 不允许只写“待补”。

4. **状态与备注必须可追踪**
   - “当前状态”应反映整体成熟度；
   - “备注”用于说明镜像、迁移、白名单、废弃项等。

## 5. 建议后续执行顺序

1. 先按 63 个域/子域列出清单；
2. 再逐域补全 §0～§11；
3. 再按“待补文档区域”分配到对应文档目录；
4. 最后统一验收与归档。
