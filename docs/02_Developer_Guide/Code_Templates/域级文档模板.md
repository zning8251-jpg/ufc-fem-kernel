## 10. 填写规范

1. 使用 UFC 统一命名，域名称、层级名称、能力名称保持前后一致。
2. 描述应以事实为主，避免引入未确认实现细节。
3. 如存在跨域依赖，需明确上游、下游与兼容约束。
4. 如存在测试、审查或守卫结果，需在追溯区建立可回溯对应关系。
5. 模板可直接复制用于 63 个域中的任一域级文档。
