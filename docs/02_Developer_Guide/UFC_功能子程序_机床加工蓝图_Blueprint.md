# UFC_中层架构件_功能子程序终极蓝图_Blueprint (v2.0 契约驱动版)

> **核心箴言**：
> 不要让“新功能怎么写”靠经验传承，必须将其固化为 **合同卡 + 契约 + 模板 + 兼容矩阵**。
> 本蓝图不仅是代码生成指令，更是 UFC 架构的“中层架构件（Middle-level Architecture Artifact）”。填写本蓝图，即等同于签署一份严格的系统工程合同。

---

## 🛠️ 第一部分：施工图与架构规范索引 (Reference Guide)
在填写工单前，请严格对照以下体系（若无特定文档，则应先补充架构设计）：
1. **域级合同与边界**：查阅 `docs/03_Domain_Pillars/` (明确L3唯一真源, L4物理计算, L5运行调度的边界)。
2. **正交兼容矩阵**：确认 Element × Section × Material × Procedure 的组合能力标签。
3. **接口签名(SIO)与四型(TYPE)**：遵守五参/六参标准，严禁传递散装变量。
4. **三重正交坐标系**：(Space-Time-Action) 必须与架构文档**绝对一致**，这是路由分发(Procedure)和后缀命名的唯一合法依据。

- **语义目标**: 计算 4 节点平面应力连续体单元的 B 矩阵及单元素积分刚度。
- **隔离边界**: 绝不计算本构方程（交给 L3_MD），绝不执行全局刚度矩阵组装（交给 L5_RT）。

### §3. SIO 接口契约与热路径声明

- **ABAQUS Hook**: 对齐 `UEL` 的局部形函数和刚度约定。

### §6. 异常诊断与观测点
- **错误出站契约**: 雅可比行列式 < 0 时，返回 `err_code=401` (网格畸变)。
- **恢复策略**: 上报 L5_RT 报错并建议重新划分网格。

### §7. 旧资产重构与守护绑定
- **旧版代码**: `UF_PH_CPS4_Stiffness.f90`
- **最小测试集**: 挂载 1x1 网格的剪切/拉伸 Patch Test (检查防锁死)。
