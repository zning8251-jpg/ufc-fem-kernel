# UFC L3/L4/L5 域柱分类重构与旧资产迁移执行总纲

## 1. 核心战略理解：什么是“域柱”切分法？

在传统的架构重构中，按“层（Layer）”横向重构往往会导致“断点”——重构了L3，但L4调不通。
您提出的**“贯通域柱、半域柱、专属层全域”**切分法，本质上是**垂直切片（Vertical Slicing）与水平黑盒（Horizontal Encapsulation）的完美结合**。

### 1.1 贯通域柱 (Full Domain Pillar)：L5 $\to$ L4 $\to$ L3 的全链路
这是 UFC 最核心的热点计算流，数据指令自上而下贯穿，物理响应自下而上冒泡。
*   **典型场景**：非线性残差与切线刚度组装 (`ItrEvl`)。
*   **链路拆解**：
    *   **L5_RT (Assembly/Solver)** 发起 `RT_Asm_Proc_GlbItrEvl` (全局组装)。
    *   **L4_PH (Element)** 响应调用 `PH_Elem_Proc_ElmItrEvl` (单元高斯积分循环)。
    *   **L3_MD (Material)** 终点承接 `MD_Mat_Proc_PtItrEvl` (单点应力与切线模量计算)。
*   **模板适配点**：严格贯彻 **5参/6参 SIO (Structured IO)**，保证跨层通讯的 `*_Arg` 极度清晰（明确 `[IN]/[OUT]`）。

### 1.2 半域柱 (Half Domain Pillar)：相邻层的局部协同
不需要全层贯通，仅在两层之间存在强耦合的局部数据交换。
*   **典型场景 A (L4 $\leftrightarrow$ L3)**：材料状态初始化 (`Init`)。L4 单元域在 0 增量步时，调用 L3 分配并初始化高斯点的塑性历史变量（`State`）。
*   **典型场景 B (L5 $\leftrightarrow$ L4)**：边界条件投影 (`LoadBC`)。L5 的外力管线将面载荷（Pressure）传递给 L4 的表面单元进行等效节点力积分。
*   **模板适配点**：设计专属的软缓存映射机制（如从 L5 的 `Glb_Ctx` 映射到 L4 的 `Elem_Ctx`）。

### 1.3 专属层全域 (Layer-Specific Domain)：层内闭环黑盒
纯粹发生在该层内部的逻辑、数学或控制算法，对外完全屏蔽实现细节。
*   **L3_MD 专属层**：各类经典本构的返回映射算子（Return Mapping）、屈服面函数（Yield Function）。放在 `MD_Mat_Core.f90` 中，纯数学操作。
*   **L4_PH 专属层**：CPE4/C3D8 等单元的等参形函数求导、B 矩阵构造、B-bar 投影防锁死。放在 `PH_Elem_Core.f90`。
*   **L5_RT 专属层**：时间步长切割自适应（`StepInc`）、收敛容差判定（`RT_Conv_Proc`）。

---

## 2. 模板体系的系统性/适配性设计准则

为了承载上述三大域柱，我们的子程序模板必须具备极强的**“结构刚性”与“内部柔性”**：

1.  **数据容器基座化 (`_Def.f90`)**：
    所有域的模板必须首抓数据。严格遵守四大类型约束（`_Desc` 静态属性, `_State` 演化历史, `_Algo` 无状态算法参数, `_Ctx` 软缓存与上下文）。严禁将其作为文件后缀。
2.  **流程管道标准化 (`_Proc.f90`)**：
    只做两件事：**路由分发（Routing）** 和 **异常冒泡（Error Handling）**。模板内通过 `SELECT CASE` 路由到对应的物理/数学核心，自身不留任何具体的数学计算公式。
3.  **核心算法纯粹化 (`_Core.f90`)**：
    这是旧资产真正的安身之所。剥离掉以前所有的全局变量、复杂传参，只允许接收纯粹的数学数组和简单类型，确保极佳的可测试性（Testability）。

---

## 3. 旧资产 (UF_* Legacy Code) 改造路线图

有了上述模板后，我们将按以下标准化流水线，像“洗煤”一样清洗现有旧代码：

*   **Step 1：拆解与映射（解耦旧状态）**
    盘点旧代码中的 COMMON 块、杂乱的 TYPE 结构体，将其“对号入座”强行填入新的 `_Desc / _State / _Ctx / _Algo` 中。
*   **Step 2：接口整形（套用 SIO 签名）**
    将旧子程序的几十个参数，全部打包为 1 个 `*_Arg`，并套用统一的 `(desc, state, algo, ctx, args)` 5参签名。
*   **Step 3：核心逻辑移栽（剥离 `UF_` 前缀）**
    将旧子程序的核心物理公式移入 `_Core.f90`。**严格执行命名铁律**：私有方法与局部变量全面摒弃 `UF_` 前缀，改用极简 `snake_case`。外部暴露模块名遵守 `[Prefix]_[Domain]_[Suffix]`。
*   **Step 4：挂载测试（闭环验证）**
    按域柱进行编译和跑通。先保证“专属层”黑盒跑通单测；再保证“半域柱”接口对接成功；最后全链路打通“贯通域柱”。

## 4. 下一步行动宣告
只要您下达具体指令（例如：“先从 L5-L4-L3 的刚度组装贯通域柱开始”或“先从 L3 的材料专属层模板开始”），我们将立即启动对应域柱的模板生成与旧资产重组！

# UFC 工业级旧资产“多轮逼近与重塑”工艺规程 (Iterative Forging SOP)

> **核心思想 (The Philosophy)**:
> 以“完美目标态”为绝对模具，视“旧有代码资产”为待加工的残缺原石。
> 不强求一次成型，允许在模具内部存在暂时的丑陋，但**对外暴露的契约必须绝对完美**。通过四次标准的加工工序，不断切削旧代码，最终逼近 **0 误差** 的工业级标准件。

---

## 🎯 工艺总则：为什么需要多轮逼近？
1. **防爆破原则**：旧资产往往高度耦合，一次性修改数据结构和底层算法容易导致科学计算结果偏离，难以排查。
2. **契约优先 (Contract-First)**：只要外部的 SIO 五参签名和内存池机制是对齐的，系统就能跑起来；内部的脏代码可以慢慢清洗。
3. **可度量原则**：每一次逼近，都必须通过“影子测试 (Shadow Test)”的精度比对（Tolerance < 1e-12）。

---

## 🏭 标准加工流水线 (The 4-Step Forging Process)

### 工序一：模具浇筑 (Phase 1 - Target Mold Casting)
* **动作**：完全无视旧代码的内部逻辑，纯粹基于《架构总纲》和《域级合同》，开动机床生成完美的壳子。
* **产出**：
  - `_Def.f90`：严格定义四大 TYPE (`Desc/State/Algo/Ctx`)，定义完美 SIO 参数。历史变量 `State` 强行声明为 `POINTER`（对接 L1 全局内存切片）。
  - `_Proc.f90`：严格实现五参分发、合规性断言与异常冒泡 (`err_code`)。
* **状态特征**：此时的 `_Core.f90` 是空的，模板编译完美通过，但缺乏物理计算能力。

### 工序二：原石粗加工与防腐桥接 (Phase 2 - Rough Machining & Bridging)
* **动作**：将旧资产 (`UF_XXX.f90`) 的核心计算逻辑整体塞入 `_Core.f90`。
* **妥协与桥接**：旧代码的参数通常长达二三十个。在 `_Proc.f90` 内部，将完美 SIO 的五参，**拆解、拼凑、强行转换**为旧代码所需的散装参数，喂给 `_Core`。
  - *极端情况*：如果旧代码完全无法拆解，则引入 `_Brg.f90`（绞杀者防腐层）进行包裹。
* **状态特征**：外表看起来是完美的 UFC 标准件，但 `_Core` 内部充斥着旧有的 `UF_` 前缀、局部大数组，甚至违规的 `ALLOCATE`。
* **质检点**：系统联调能够跑通，物理结果与旧版一致。

### 工序三：精加工与内存剥离 (Phase 3 - Precision Machining)
* **动作**：深入 `_Core.f90` 内部，执行外科手术式的“切削”。
  1. **内存剥离**：杀掉所有的 `ALLOCATE` 和局部巨型数组，将中间变量强行挂载到传入的 `_Ctx` 缓存中，将历史变量强行路由到 `_State` 切片。
  2. **命名清洗**：抹除所有 `UF_` 前缀，重构为 `snake_case`。
  3. **去全局化**：清理掉内部任何尝试访问外部全局变量的行为，将依赖全部通过 `_Desc` 或 `_Algo` 传入。
* **状态特征**：代码变得极其紧凑，符合 UFC 高性能并行和 GPU 移植的内存规范。

### 工序四：无损探伤与 0 误差收敛 (Phase 4 - Zero-Error Convergence)
* **动作**：启动“影子测试 (Shadow Test)”。
* **操作法**：
  在同一个测试驱动 (Harness) 下，用相同的输入 (`deps`, `dt` 等)，**同时跑旧版原生模块与我们精加工后的完美模块**。
  提取输出量（如 `stress`, `Jacobian`），逐个元素计算 L2 范数误差。
* **质检点**：误差绝对值 `< 1.0E-12`。如果超差，打回工序三重新检查数据流。
* **终局**：一旦逼近 0 误差，彻底删除旧资产对照组，该模块正式成为 **“UFC 完美标准件”**。

# UFC 各域功能子程序模板生成标准操作流程 (SOP-TGT)

为了确保未来所有生成的各层（L1~L6）、各域（Domain）的功能子程序模板都能**精准无误地契合 UFC 全景架构（极简三段式、5/6 参 SIO、四大类型约束）**，我们为您制定了以下专属的提示词框架（Prompt Protocol）与执行流水线。

---

## 1. 您给我的专属提示词框架 (Prompt Template)

每当您需要我为您开发某个特定域、某个具体功能（如：接触对摩擦、流体单元、时间步切割等）的标准化 `_Def / _Proc / _Core` 模板时，**您只需直接向我发送或稍微修改以下这段提示词：**

```markdown
【启动模板生成指令：UFC-Template-Gen】
1. 目标层与域 (Target Layer & Domain)：
   - (例如：L4_PH_Element 或 L3_MD_Contact 或 L5_RT_StepDriver)
2. 物理/算法特性 (Target Feature)：
   - (例如：实现 2D 平面应力四节点单元 CPS4 的刚度组装与 B-bar 防锁死)
3. 跨层联动需求 (Inter-layer Dependencies)：
   - 向下调用：(例如：需要调用 L3_MD_Material 计算应力)
   - 向上返回：(例如：向 L5_RT_Assembly 返回单元残差和切线刚度)
4. 挂载的旧资产参考 (Legacy References)：
   - (例如：请参考现有旧代码 `ufc_core/UF_PH_CPS4.f90` 中的几何拓扑和插值函数)
```

## 2. 接收指令后，我的标准执行流 (AI Execution Pipeline)

当我收到上述指令后，我将严格按照以下 **4 步流水线** 为您生成或改写代码，且每一步都带有严苛的架构红线审查：

### ⚔️ Step 1: 打造数据底座 (`[Prefix]_[Domain]_Def.f90`)
*   **动作**：强制切分您的旧代码常量与变量，填入 `_Desc` (静态属性)、`_State` (演化历史)、`_Algo` (算法开关) 和 `_Ctx` (软缓存)。
*   **审查**：绝不能出现 `[Type].f90` 这种违规后缀；在文件末尾定义包含明确 `[IN]/[OUT]` 注释的 Principle #14 SIO 参数包（`*_Arg`）。

### ⚔️ Step 2: 铺设路由与跨层管线 (`[Prefix]_[Domain]_Proc.f90`)
*   **动作**：生成标准的 `(desc, state, algo, ctx, args)` 5/6 参签名接口。利用 `SELECT CASE` 建立针对不同功能（如不同单元类型、不同求解器策略）的路由分发。
*   **审查**：此文件中**严禁出现任何具体的物理或数学计算公式**；严查异常冒泡机制（`err_code`）是否贯通。

### ⚔️ Step 3: 封装纯粹物理/数学黑盒 (`[Prefix]_[Domain]_Core.f90`)
*   **动作**：将旧资产中的真核逻辑（如雅可比求逆、形函数求导、本构返回映射）全盘移入。剥离全局 COMMON 块，使其变成纯净的内存操作。
*   **审查**：**核心铁律：内部彻底抹杀 `UF_` 前缀**，局部变量必须使用极简蛇形命名 (`snake_case`)，不带任何赘余模块名。

### ⚔️ Step 4: 防腐层或操作子域的挂载（可选 `_Brg.f90` 或 `_Ops.f90`）
*   **动作**：如果该功能涉及调用外部数学库（如 AMG, PETSc），我会为您生成防腐桥接层 `_Brg.f90`；如果涉及矩阵操作或底层张量工具，则生成 `_Ops.f90`。
