# UFC 架构设计文档指导手册

> 本手册对 **docs** 与 **六层架构拆分** 按目录/章节/内容进行分类梳理，明确权威来源与阅读顺序，标注冗余与可删除项，供架构设计时作为**单一指导手册**使用。  
> 与 [00-知识体系与设计框架总览](./00-知识体系与设计框架总览.md) 配合：总览为知识体系入口与设计净资产；本手册为**文档体系分类与使用说明**。

---

## 一、手册定位与使用方式


| 对象     | 用途                               |
| ------ | -------------------------------- |
| 架构设计   | 按「设计活动」快速找到必读文档与权威来源，避免在冗余文档中迷失。 |
| 新人/LLM | 按「目录—章节—内容」理解文档树，按手册给出的阅读路径一次走完。 |
| 维护者    | 按「内容分类与清理建议」做归档、合并、删除，保持文档集可维护。  |


**使用方式**：  

- 做**架构/层/域设计** → 看 **三、按设计活动的章节导航** + 总览「软件系统架构设计必读路径」。  
- 做**命名/契约/数据结构/实施/验证** → 看 **三** 对应小节与 **四、内容分类表**。  
- 做**文档清理** → 看 **五、冗余清理与维护**，按「已归档 / 建议合并或删除」执行。

## 二、文档体系总览（目录—角色—设计活动）

### 2.1 顶层结构

```
ufc_core/docs/                # 文档根：入口、六层架构拆分、契约、技能
├── 00-知识体系与设计框架总览.md   # 唯一知识体系入口
├── index.md                  # 按类型列出的文档清单
├── UFC架构设计文档指导手册.md    # 本手册
├── 命名与契约类 *.md
├── 数据结构与类型类 *.md
├── 理论与手册类 *.md
├── skills/                   # 域级技能模板（如 TL/UL 链路）
├── archive/                  # 已归档冗余/历史，不作设计依据
│
└── 六层架构拆分/              # 已迁入 docs：架构定义、总纲、理论、实施、各层拆解
    ├── 00-总纲/              # 当前有效：蓝图、规范、契约、99-Skill
    │   └── archive/          # 历史轮次（10～28），仅参考
    ├── 01-理论基础/
    ├── 02-架构定义/           # 六层、Context、Bridge、数据流、错误、验证
    ├── 03-实施路线/
    ├── 04-六层架构拆解/       # L1～L6 各层总览与专题
    ├── 05-工程规范/
    └── 06-附录/
```

### 5.3 维护规则

- **新增文档**：在总览「一、知识体系结构」与本手册「四、内容分类表」中登记；权威文档保持原位，不放入 archive。
- **冗余判定**：与 02-架构定义、00-总纲、docs 结构体规划/命名契约 重复或已被取代 → 移入对应 archive 或合并后删除。
- **历史轮次**：多轮「完成报告/深度设计/规划」类 → 放入 00-总纲/archive，总纲内只保留当前有效蓝图与规范。

## 六、设计文档编写与更新约定

- **单一权威**：每个主题（架构、命名、数据结构、关键字、域级规范）只保留一个权威文档或最小集合，其余引用或链接。
- **总览为入口**：新增权威来源时同步更新 00-知识体系与设计框架总览 的「一、知识体系结构」与「六、概念→文档」。
- **本手册同步**：目录结构、分类表、清理建议变更时，同步更新本手册第二、四、五章。
- **避免再冗余**：新写「架构方案/规划/计划」前先查是否与 02-架构定义、00-总纲、03-实施路线 重复；若为阶段性报告，计划放在 00-总纲/archive 或单独「计划」目录，不替代总纲正文。
