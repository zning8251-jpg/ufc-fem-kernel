## 1. 核心底座：哲学总纲与数据真源 (Docs 01-10, 16, 21)

这是 UFC 能否脱离“手工作坊”，走向工业软件的命脉。

- **六层四类四链**：确立了 `L1~L6` 层级，死锁了四大类型后缀 `_Desc`, `_State`, `_Ctx`, `_Algo`（且明令禁止以此作为 `.f90` 后缀），打通了理论、逻辑、计算、数据四条链路。
- **关系型扁平化数据真源**：废弃了 ABAQUS 逻辑层面上 6 层树状物理嵌套！采用**外键 ID 关联的扁平池**，物理 TYPE 嵌套绝对不超过 3 层。
- **三级内存与软缓存**：外存（落盘） $\to$ 大内存池（DDR, 扁平数组） $\to$ 软缓存（L1 Cache）。`**Ctx` 和 `Args` 的本质就是软件定义的 L1 高速缓存块**，实现光速计算！
- **SIO 结构化 IO**：严禁面条参数。核心子程序死锁为 5/6 参签名（`desc, state, algo, ctx, [com], args`）。

## 2. 运行时齿轮：时空动三维正交与压缩命名 (Docs 11-15)

这是 UFC 运转的精密时钟，彻底消灭了旧版“时相×动作”的维度坍缩和歧义。

- **X 轴（空间 Where）**：`Glb` (全局), `Reg` (区域), `Elm` (单元), `Pt` (积分点), 以及为 DG 准备的 `**Fac` (面)**。
- **Y 轴（时相 When）**：宏观 Step/Inc $\times$ 微观 Init/Prd/Itr/Cmt/Rbk。
- **Z 轴（动作 What）**：`Pop` (捞取), `Evl` (测算), `Asm` (组装), `Map` (映射), `Exp` (输出)。
- **3-LAR 命名法**：所有的子程序被强制定轨，例如 `_Proc_ElmItrEvl`，使得千万行代码如臂使指。

## 3. 宏观路由与极端的非线性防线 (Docs 17-20, 22-24)

这是超越 ABAQUS 等顶级商业软件的武装。

- **双域三维正交路由**：
  - 司令部路由：`[物理场] × [求解器] × [耦合形式]`
  - 后勤适配路由：`[单元 L4] × [截面 Section] × [材料 L3]`
- **四大终极防线**：
  1. **自动微分（AD）混合引擎**：零成本生成绝对精确的切线刚度 $D_{ep}$。
  2. **B-bar 与线搜索**：破除不可压锁死，抑制弧长法发散。
  3. **矩阵冷热探针与热切换**：对称 $\to$ 非对称矩阵，静态 $\to$ 动态求解器自动切换。
  4. **增广拉格朗日**：平滑接触震荡，替代生硬的罚函数。
- **求解器外挂战略**：L2_NM 绝对不造轮子。直接法接驳 Intel MKL / MUMPS；亿级超大规模迭代接驳 PETSc + AMG（代数多重网格）。

## 4. 大一统境界：PDE 统一基座与算子注入 (Docs 25-26)

这是 UFC 名字中 UniField 的终极体现，降维打击传统 CFD。

- **场注册表（Field Registry）**：破除硬编码 $(u,v,w)$。系统初始化时动态注册温度（Heat）、速度与压力（CFD，不等阶）、电磁势（EM，边单元）、孔隙水压（Seepage），由引擎自动划分多场拓扑方程块。
- **算子注入（Operator Injection）**：底层单元只提供纯 Galerkin 残差。遇到对流方程，由引擎动态向挂载点注入 **SUPG/GLS/VMS** 稳定化算子，根除对流震荡。
- **算子分裂管线（Fractional Step）**：不头铁去强解病态鞍点矩阵！将 CFD 拆分为动量步（非对称求解）、压力泊松步（对称 AMG 秒杀）和速度修正步，完美破局收敛性。
