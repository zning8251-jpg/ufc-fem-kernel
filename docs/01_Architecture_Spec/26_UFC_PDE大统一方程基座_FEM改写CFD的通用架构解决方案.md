## 1. 核心解法一：弱形式与稳定化的“算子注入式（Operator Injection）”设计

传统软件往往把稳定化项（如 SUPG）硬编码在流体单元里。在统一基座中，这会导致代码极度冗余。

**UFC 通用解决方案：变分多尺度（VMS）大一统与算子注入**

- **物理剥离**：在 `L4_PH` 层，所有的单元只负责提供**标准伽辽金（Standard Galerkin）**的基础残差 $R_{Gal}$ 和雅可比 $K_{Gal}$。
- **算子注入点**：在 `_Proc_ElmItrEvl` 的尾部，预留一个钩子（Hook）：`Stabilization_Operator`。
- **动态挂载**：
  - 如果是固体力学，该挂载点为空。
  - 如果是 CFD，求解器动态向挂载点注入 **SUPG/GLS 算子**，或者最高维度的 **VMS（变分多尺度）算子**。
  - 稳定化算子通过提取当前单元的对流速度和网格特征尺度 $\tau$，**在原本的 $R_{Gal}$ 上自动附加微扰项 $\Delta R_{stab}$**。
- **优势**：通过这种“算子注入”，UFC 可以在不改动底层单元逻辑的前提下，一秒钟从普通流体切换为支持大涡模拟（LES）的高雷诺数流体！

## 2. 核心解法二：混合自由度拓扑与 LBB 鞍点破局

CFD 要求速度与压力必须满足 LBB（inf-sup）条件，这在数据结构上要求速度用二次节点（如 9 节点），压力只用一次节点（如 4 节点）。

**UFC 通用解决方案：场注册表（Field Registry）与非均匀自由度池**

- **打破固化节点**：UFC 的 `g_ufc_global` 彻底废弃每个节点固定 $X/Y/Z$ 自由度的概念。
- **场声明驱动**：在初始化时，应用层 `L6_AP` 声明：
`Call RegisterField('Velocity', VECTOR, Order=2)`
`Call RegisterField('Pressure', SCALAR, Order=1)`
- **拓扑生成器**：UFC 底层的方程编号器（Equation Mapper）自动遍历网格，为所有的角点和边中点分配速度方程 ID，但**只为角点**分配压力方程 ID。
- **结果**：UFC 极其优雅地原生支持了 Taylor-Hood 元和 MINI 元。无论是解流体的鞍点，还是解电磁学的混合势，数据容器游刃有余。

## 3. 核心解法三：连续与间断的统一（DG 与 CIP 框架）

您提到用 DG-FEM（间断伽辽金）替代黎曼求解器，这是高阶 CFD 和激波捕捉的命脉。

**UFC 通用解决方案：“面域（Face Domain）”的一等公民化**

- **三维空间轴的补全**：UFC 架构的空间轴原本是 `Global $\to$ Elem $\to$ Point`。现在正式引入第四维：`**Face` (单元交界面)**。
- **DG 求解管线**：对于高马赫数可压流，单元内部调用 `_Proc_ElmItrEvl` 算体积分；单元交界面触发 `_Proc_FaceItrEvl`。在面上，UFC 自动抓取左右单元的数据，计算精确的数值黎曼通量，并组装非对角块。
- **CIP 妥协路线（连续内罚）**：由于纯 DG 自由度爆炸，UFC 同样支持 CIP 方法。物理场保持连续，但在 `Face` 域上针对梯度的跳跃计算额外的罚函数。这是 UFC 特有的低成本、高鲁棒激波捕捉折中方案！

## 4. 核心解法四：强非线性与病态矩阵的管线切割（算子分裂）

直接用全牛顿法解带 SUPG 的 Navier-Stokes 方程，由于强对流和鞍点极难收敛。

**UFC 通用解决方案：L5_RT 的“伪时间与算子分裂（Operator Splitting）”状态机**

- **管线 A：伪时间推进（Pseudo-Transient Continuation）**
将对角线上附加质量项 $\frac{M}{\Delta \tau}$。当残差很大时，$\Delta \tau$ 极小，保证对角线占优，强行稳住初值；当残差下降，$\Delta \tau \to \infty$，自动恢复为全牛顿二次收敛。
- **管线 B：算子分裂 / 投影法（Fractional Step / Projection Method）**
这是超越直接求解的无敌策略。把巨大的多场矩阵拆散！
  1. **对流扩散步**：只组装动量方程（不带压力），使用 SUPG 算出一个中间速度场（求解非对称但在对角线占优的矩阵）。
  2. **泊松投影步**：组装压力的拉普拉斯方程，求解压力增量。这里矩阵是**绝对对称正定**的！直接调用极其成熟的 AMG（代数多重网格）秒杀。
  3. **速度修正步**：用压力梯度修正流场，满足质量守恒。
- **优势**：通过 L5_RT 的状态机拆分，UFC 完美避开了在求解极其病态的鞍点巨型矩阵时的发散风险，将 CFD 转化为连续求解几个简单矩阵的过程！

## 5. 通用方案总结：UFC 大统一基座的运作流

如果今天我们要在 UFC 上求解一个**同时带有固体大变形、高雷诺数流体流动的多场问题**，系统的运转流将是这样的：

1. **L6 场注册**：注册 `Disp(固)`, `Vel(流,二次)`, `Press(流,一次)`。
2. **L5 路由**：生成算子分裂管线。定义伪时间步长自适应策略。
3. **L4 遍历**：
  - 遍历流体单元：计算标准伽辽金，**注入 SUPG / VMS 稳定化算子**。
  - 遍历固体单元：计算 B-bar 防锁死。
  - 遍历边界：触发 `Face` 计算黎曼通量或边界层条件。
4. **L3 本构**：自动微分（AD）引擎提供流体偏应力和固体切线刚度。
5. **L2 数学库**：动量步交给非对称迭代（BiCGSTAB + ILU），压力步交给对称多重网格（CG + AMG）。
