## 1. 版本号规则

### 1.1 语义化版本

**格式**: `主版本号.次版本号.修订号`（Semantic Versioning）

**规则**:

- **主版本号**: 不兼容的 API 变更
- **次版本号**: 向后兼容的功能新增
- **修订号**: 向后兼容的问题修复

**示例**:

- `v1.0.0` - 初始发布
- `v1.1.0` - 新增功能（向后兼容）
- `v1.1.1` - 问题修复
- `v2.0.0` - 重大变更（不兼容）

### 1.2 预发布版本

**格式**: `主版本号.次版本号.修订号-<标识>`

**标识**:

- `alpha` - 内部测试版本
- `beta` - 公开测试版本
- `rc` - 发布候选版本

**示例**:

- `v1.0.0-alpha.1` - 内部测试版本
- `v1.0.0-beta.1` - 公开测试版本
- `v1.0.0-rc.1` - 发布候选版本

## 2. 向后兼容性保证

### 2.1 兼容性级别

**完全兼容**:

- API 接口不变
- 数据格式不变
- 行为不变

**部分兼容**:

- API 接口扩展（新增参数可选）
- 数据格式扩展（向后兼容）
- 行为增强（不影响现有功能）

**不兼容**:

- API 接口变更（删除、重命名）
- 数据格式变更（无法读取旧格式）
- 行为变更（影响现有功能）

### 2.2 兼容性保证期限

**保证期限**: 至少支持 **2 个大版本**的向后兼容

**示例**:

- v2.0 必须能读取 v1.x 的 .inp 文件
- v3.0 必须能读取 v2.x 的 .inp 文件
- v3.0 可以不支持 v1.x（已过 2 个大版本）

## 4. 不兼容变更列表

### 4.1 v2.0 不兼容变更

#### API 变更


| 旧接口                  | 新接口                        | 迁移方法   |
| -------------------- | -------------------------- | ------ |
| `UF_Material_Init`   | `MD_Material_Desc_Init`    | 使用迁移工具 |
| `VD_Part_Create`     | `MD_Part_Desc_Init`        | 使用迁移工具 |
| `CF_Element_Compute` | `PH_Elem_ComputeStiffness` | 使用迁移工具 |


#### 数据格式变更


| 旧格式         | 新格式         | 迁移工具                      |
| ----------- | ----------- | ------------------------- |
| `.inp` v1.0 | `.inp` v2.0 | `convert_inp_v1_to_v2.py` |
| `.odb` v1.0 | `.odb` v2.0 | `convert_odb_v1_to_v2.py` |


#### 行为变更


| 旧行为               | 新行为                    | 影响         |
| ----------------- | ---------------------- | ---------- |
| 默认精度 `REAL(8)`    | 默认精度 `wp`（可配置）         | 可能影响数值结果   |
| 错误处理 `WRITE(*,*)` | 错误处理 `ErrorStatusType` | 需要更新错误处理代码 |

## 5. 迁移工具

### 5.1 格式转换工具

#### convert_inp_v1_to_v2.py

**用途**: 将 .inp 文件从 v1.0 转换为 v2.0

**使用方法**:

```bash
python tools/convert_inp_v1_to_v2.py input_v1.inp output_v2.inp
```

**功能**:

- 转换命令格式
- 更新关键字
- 更新数据格式

**示例**:

```bash
# 转换单个文件
python tools/convert_inp_v1_to_v2.py old_model.inp new_model.inp

# 批量转换
python tools/convert_inp_v1_to_v2.py --batch input_dir/ output_dir/
```

#### convert_odb_v1_to_v2.py

**用途**: 将 .odb 文件从 v1.0 转换为 v2.0

**使用方法**:

```bash
python tools/convert_odb_v1_to_v2.py input_v1.odb output_v2.odb
```

### 5.2 API 迁移工具

#### migrate_api_v1_to_v2.py

**用途**: 自动迁移代码从 v1.0 API 到 v2.0 API

## 6. 版本发布策略

### 6.1 版本发布周期

**主版本**: 每年 1 次（重大变更）  
**次版本**: 每季度 1 次（功能新增）  
**修订版本**: 每月 1 次（问题修复）

### 6.2 版本支持策略

**支持期限**:

- **当前版本**: 完全支持
- **上一版本**: 完全支持
- **更早版本**: 仅安全修复

**示例**（当前版本 v2.0）:

- v2.0: ✅ 完全支持
- v1.2: ✅ 完全支持
- v1.1: ⚠️ 仅安全修复
- v1.0: ❌ 不再支持

### 6.3 废弃通知策略

**废弃流程**:

1. **通知**: 在次版本中标记为废弃（Deprecated）
2. **警告**: 在修订版本中发出警告
3. **移除**: 在下个主版本中移除

**示例**:

- v1.1: `UF_Material_Init` 标记为废弃
- v1.2: `UF_Material_Init` 发出警告
- v2.0: `UF_Material_Init` 已移除
