# 第 13 件：Test (单元测试与桩代码件) 设计规范与模板

> **文档位置**：`UFC/docs/05_Project_Planning/PPLAN/13_高阶组件技术规范_Spec/12_第13件_Test_单元测试与桩代码件_设计规范与模板.md`  
> **所属套件**：全景套件 v3.0 第 13 件  
> **目标**：在不启动庞大 L6 框架和 MPI 集群的前提下，实现 L4/L5 核心数学逻辑的微型测试、分片测试 (Patch Test) 和自动化验证 (V&V)。

---

## 1. 核心目标与技术红线

### 1.1 核心目标
1. **测试隔离 (Test Isolation)**：单元测试不需要读取 `.inp` 文件，直接构造（Mock）内存中的 `_Desc` 和 `_State` 就能触发真正的物理计算引擎。
2. **桩代码注入 (Mock Injection)**：通过替身模块切断上下游依赖，确保单元测试是原子的。

### 1.2 架构红线 (Red Lines)
- **物理隔离**：测试件和桩代码的模块，**绝对不允许**存在于 `ufc_core/` 源代码主干目录中。它们必须存放在独立的 `Tests/` 或 `Mock/` 工程目录下。
- **无状态验证**：每个 Test Case 执行前后，系统的全局状态不能受污染，确保用例随时可以乱序执行。

---

## 2. 核心架构时序与机制

1. **[Setup阶段]**：在 `Tests` 目录下，直接实例化一个极其迷你的 `_Desc` 数组（例如只包含一个八节点六面体的坐标）。初始化一个空的 `_State` 和包含目标容差的 `_Algo`。
2. **[Mock桥接]**：如果不测全局装配，可以通过 `Mock_Bridge` 把这个单独的单元伪装成全工程。
3. **[Run核心]**：调用 `PH_Elem_C3D8_Eval` 执行计算。
4. **[Assert阶段]**：比较产出的 `_State%stress` 和雅可比矩阵是否与 NAFEMS 标准答案 (硬编码常量) 在 `1.0E-6` 内吻合。

---

## 3. 伪代码模板与合同定义

```fortran
!=============================================================================
! MODULE: Test_PH_ElemC3D8
! 描述: 针对 C3D8 单元内核的分片微型测试
! 存放路径: UFC/Tests/PH_Elem/Test_PH_ElemC3D8.f90
!=============================================================================
MODULE Test_PH_ElemC3D8
    ! 仅依赖生产核心与四型定义
    USE PH_Elem_Def
    USE PH_ElemC3D8
    IMPLICIT NONE
    PRIVATE

    PUBLIC :: Run_Patch_Test_C3D8

CONTAINS

    !> C3D8 单单元分片拉伸测试
    SUBROUTINE Run_Patch_Test_C3D8(pass)
        LOGICAL, INTENT(OUT) :: pass
        
        ! 本地栈上直接分配，摒弃所有前处理解析
        TYPE(PH_Elem_Desc)  :: mock_desc
        TYPE(PH_Elem_State) :: mock_state
        TYPE(PH_Elem_Algo)  :: mock_algo
        TYPE(PH_Elem_Ctx)   :: mock_ctx
        INTEGER :: status
        
        ! 1. Mock Setup (构造标准立方体)
        ALLOCATE(mock_desc%nodal_coords(3,8))
        mock_desc%nodal_coords(:,1) = [0.0d0, 0.0d0, 0.0d0]
        mock_desc%nodal_coords(:,2) = [1.0d0, 0.0d0, 0.0d0]
        ! ... 初始化 8 个节点 ...
        
        ALLOCATE(mock_ctx%local_stiffness(24,24))
        
        ! 2. 执行计算
        CALL PH_Elem_C3D8_Eval(mock_desc, mock_state, mock_algo, mock_ctx, status)
        
        ! 3. Assert (验证雅可比行列式或单元刚度迹)
        pass = .TRUE.
        IF (status /= 0) pass = .FALSE.
        
        ! 验证刚度矩阵对角线元素必须全正
        IF (ANY(DIAGONAL(mock_ctx%local_stiffness) <= 0.0d0)) THEN
            pass = .FALSE.
            PRINT *, "Test Failed: Negative stiffness detected."
        END IF
        
    END SUBROUTINE Run_Patch_Test_C3D8

END MODULE Test_PH_ElemC3D8
```

## 4. 合同检验点 (Checklist)
1. 检查 CMake/Makefile 脚本，是否在发布 (Release/Production) 的二进制中，彻底排除了 `Test_` 或 `Mock_` 开头的模块？
2. 测试代码能否绕过主执行流程（即不走 `L6_AP -> L5_RT` 的复杂调用链），直接对 `L4_PH` 暴露的接口实现原子调用？（如果不能，说明强耦合）。
3. 测试中 `_Ctx` 的临时工作区是否在测试结束后被正确清理和释放，没有导致测试链发生内存泄漏？
