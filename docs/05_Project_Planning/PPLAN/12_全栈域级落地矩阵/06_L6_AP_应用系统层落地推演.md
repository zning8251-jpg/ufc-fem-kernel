# L6_AP 应用与系统接口层落地推演

> **文档位置**：`docs/05_Project_Planning/PPLAN/12_全栈域级落地矩阵/06_L6_AP_应用系统层落地推演.md`

## 总体定位
**L6_AP (Application & System)** 是 UFC 暴露给最终用户、外部脚本系统、甚至 AI 代理的最外层表皮。在这一层，必须拦截一切非法输入、越界调用，并将用户语言（如 INP 指令、Python 调用）转化为 UFC 底层的 `L3_MD` 领域模型，最后转交 `L5_RT` 执行。

---

### 域级编码： `AP_Facade` (通用门面/解析域)

#### 1. 核心意图与完成体目标
- **设计意图**：作为输入解析器（如解析 Abaqus INP 文件），负责将基于字符串的配置反序列化。
- **完成体目标**：隔离“语法解析”与“物理模型构建”。语法解析失败在这一层直接弹出，绝不会在底层计算中发生“找不到材料”等问题。

#### 2. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `AP_Facade_Def.f90` | 命令行参数结构、解析状态码 |
| _Ctx     | `AP_Facade_Ctx.f90` | Parser 上下文（当前行号、词法分析缓冲） |
| _Algo    | `AP_Facade.f90`（旧称 `AP_Facade_Algo.f90`）| 字符串切分、正则匹配、关键字提取算法 |
| _Brg     | `AP_Facade_Brg.f90` | 向外提供主入口如 `Facade_ParseINP` |
| _Populate| `AP_Facade_Pop.f90` | 极其重要：调用 L3/L4 的 `_Populate` 生成对应的模型描述对象 |

#### 3. 打通思路与底层代码推断
```fortran
SUBROUTINE AP_Facade_BuildModel(filename, model_ctx, err_stat)
    ! 1. 解析词法
    ! 2. 调用下游 Builder 生成对象
    CALL L3_Material_Populate(..., err_stat)
    IF (IF_Error_IsBad(err_stat)) RETURN
END SUBROUTINE
```

### 域级编码： `AP_Python` (Python/C ABI 绑定域)

#### 1. 核心意图与完成体目标
- **设计意图**：提供兼容 ISO_C_BINDING 的内存布局，供 Python 经由 `ctypes` 或 `cffi` 调用 UFC。
- **完成体目标**：支持在 Python 端编写高级优化循环，并在每一步提取 UFC 的网格坐标与应力场。

#### 2. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Brg     | `AP_Python_Brg.f90` | 只包含 `BIND(C, name="...")` 的标准导出函数。 |
| _Dump    | `AP_Python_Dump.f90`| 用于验证 C 结构体对齐的工具代码 |

#### 3. 底层代码推断
任何导出的接口必须屏蔽 Fortran 特性（如假定形状数组、派生类型方法），全部降级为 C 指针和标量：
```fortran
! BIND(C) 标准出口
SUBROUTINE UFC_SubmitJob_C(inp_path_c, status_code) BIND(C, name="UFC_SubmitJob")
    USE, INTRINSIC :: ISO_C_BINDING
    TYPE(C_PTR), VALUE :: inp_path_c
    INTEGER(C_INT), INTENT(OUT) :: status_code
    
    ! 内部转换为 Fortran 字符串，调用 RT_Job_Submit
END SUBROUTINE
```

### 域级编码： `AP_Monitor` (系统监控与遥测域)

#### 1. 核心意图与完成体目标
- **设计意图**：汇集底层 L1_IF_Logger 与 L5_RT 的运行状态，向上层 UI 或 AI Agent 输出结构化状态。
- **完成体目标**：类似于 Abaqus 的 `.sta` 文件生成器，或者供 REST API 读取的 JSON 状态吐出。

#### 2. 物理推演
使用 `AP_Monitor.f90`（旧称 `AP_Monitor_Algo.f90`） 每 N 步汇聚一下牛顿法的迭代残差、时间步长、L-BFGS 重启次数，统一格式化写出。这为未来 **AI 接管自适应参数** 提供了唯一的“传感器”数据源。
