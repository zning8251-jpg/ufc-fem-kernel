## 总体定位
**L4_PH (Physics Component)** 聚焦于有限元方法的几何和数学组件。涉及节点、自由度、形函数推导、单元局部积分。本层只看到“单个单元/节点”，不具有全局求解视野（不直接看到全局矩阵，只吐出 $K_e$ 和 $R_e$ 给上层组装）。

### 域级编码： `PH_Node` 与 `PH_DOF` (节点与自由度管理域)

#### 1. 核心意图与完成体目标
- **设计意图**：剥离坐标存储（Node）与物理状态未知量映射（DOF）。节点只记录空间拓扑，DOF记录解向量到全局矩阵的索引映射（Equation ID）。
- **完成体目标**：支持多物理场（比如节点不仅有U1,U2,U3，还可能有温度T，甚至孔隙水压Pw），支持激活/杀死节点的自由度缩聚。

#### 2. 合同卡设计
- **输入依赖**：边界条件（BC）用于标记固定自由度；分析类型（决定分配哪些自由度）。
- **输出写回**：生成全局的方程映射表（Equation Mapper），为组装CSR和解向量更新提供 O(1) 索引。

#### 3. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `PH_DOF_Def.f90`   | `DOF_U1`, `DOF_U2`, `DOF_T`, `DOF_ACTIVE`, `DOF_FIXED` |
| _Desc    | `PH_Node_Desc.f90` | 初始坐标 `coord_0(:,:)`，节点标签映射 |
| _State   | `PH_Node_State.f90`| 当前坐标 `coord_t`，当前位移 `u`, 速度 `v` |
| _Algo    | `PH_DOF.f90`（旧称 `PH_DOF_Algo.f90`）  | DOF 映射算法（将活性节点自由度编号为 $1 \sim N_{eq}$） |
| _Brg     | `PH_Node_Brg.f90`  | 获取某节点指定 DOF 方程号的查询门面 |

#### 4. 打通思路与底层代码推断
```fortran
! 底层代码推断：自由度映射与获取
PURE FUNCTION PH_DOF_GetEqId(dof_map, node_idx, dof_type) RESULT(eq_id)
    ! dof_map(dof_type, node_idx) 存储了全局未知量序号
    ! 若该自由度被固定或不支持，返回 0 甚至负数处理罚函数
END FUNCTION
```

### 域级编码： `PH_Shape` 与 `PH_Integration` (形函数与积分域)

#### 1. 核心意图与完成体目标
- **设计意图**：将等参元（Isoparametric）的形函数计算完全解耦，形函数不知道自己算的是位移还是温度。
- **完成体目标**：支持极高效率的 $N_i$ 和偏导数 $\frac{\partial N_i}{\partial \xi}$ 计算。

#### 2. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Desc    | `PH_Integration_Desc.f90`| 高斯点局部坐标与权重（$\xi, \eta, \zeta, w$） |
| _Algo    | `PH_Shape.f90`（旧称 `PH_Shape_Algo.f90`）| 提供诸如 `Shape_C3D8`, `Shape_CPS4` 的具体纯函数实现 |
| _Brg     | `PH_Shape_Brg.f90` | 基于单元拓扑类型自动分发 |
