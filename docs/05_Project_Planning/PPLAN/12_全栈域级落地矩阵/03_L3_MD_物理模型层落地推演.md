# L3_MD 物理模型层落地推演

> **文档位置**：`docs/05_Project_Planning/PPLAN/12_全栈域级落地矩阵/03_L3_MD_物理模型层落地推演.md`

## 总体定位
**L3_MD (Model Definitions)** 是业务属性的静态载体（相当于 Abaqus 的 `*MATERIAL`, `*SOLID SECTION`, `*BOUNDARY`）。它的核心特征是“装配好之后，在计算（RT）期间原则上只读（Desc）”，只有与历史强相关的对象会有 `State`。

---

### 域级编码： `MD_Material` (材料域 - 已在11专项深度固化)
*(请详见：`docs/05_Project_Planning/PPLAN/11_闭环落地专项/03_L3_MD_Material_闭环落地设计.md`)*
这里仅重申：材料域被剥离为 `_Desc` (常数如 E, nu) 与 `_State` (历史如塑性应变)，依靠 `_Algo` (`J2_Plasticity`) 实现 `sigma` 的更新与切线算子推导。

---

### 域级编码： `MD_Section` (截面域)

#### 1. 核心意图与完成体目标
- **设计意图**：将材料（Material）与几何属性（厚度、积分点规则）绑定，构成直接供给 L4_PH_Element 的完全体配置。
- **完成体目标**：支持壳厚度积分、平面应力/应变标识识别，隔离单元底层与全局材料库的直接耦合。

#### 2. 合同卡设计 (Domain Contract)
- **输入依赖**：绑定的 `MD_Material_Desc` 指针，宏观拓扑标识。
- **输出写回**：合成 `ElemCtx` 给 Element。

#### 3. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `MD_Section_Def.f90` | 截面类型枚举 (`SEC_SOLID`, `SEC_SHELL`) |
| _Desc    | `MD_Section_Desc.f90`| 厚度(t)、积分法则(Gauss/Simpson)、材料指针 |
| _Container|`MD_Section_Domain.f90`| 管理全模型的截面字典库 |
| _Brg     | `MD_Section_Brg.f90` | 根据 Section 类型派遣不同的积分策略 |

### 域级编码： `MD_LoadBC` (载荷边界域)

#### 1. 核心意图与完成体目标
- **设计意图**：统一管理狄利克雷边界（Displacement）与诺依曼边界（Force/Pressure）。
- **完成体目标**：在每一分析步（Step）或增量步（Inc），快速根据时间幅值曲线（Amplitude）提取当前边界因子，提供给 `RT_Assembly` 降维或加罚。

#### 2. 合同卡设计 (Domain Contract)
- **输入依赖**：当前总时间/步时间、幅值曲线对象、节点自由度映射。
- **输出写回**：生成目标缩放后的 `LoadBC_State` 载荷向量/位移罚函数列表。

#### 3. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `MD_LoadBC_Def.f90` | `BC_DISP`, `BC_FORCE`, `BC_PRESSURE` |
| _Desc    | `MD_LoadBC_Desc.f90`| 施加集（NodeSet/SurfSet）、自由度索引、名义幅值 |
| _State   | `MD_LoadBC_State.f90`| 当前时步评估出的实际缩放幅值 |
| _Brg     | `MD_LoadBC_Brg.f90` | 为 L5_RT 暴露 `Evaluate_BC(time, dt)` 方法 |

#### 4. 打通思路与底层代码推断
```fortran
! 边界值计算闭环推断
SUBROUTINE MD_LoadBC_Evaluate(bc_desc, amp_desc, time, current_val)
    ! 根据 amplitude(time) * bc_desc%nominal_value 计算当前施加值
    current_val = MD_Amplitude_Eval(amp_desc, time) * bc_desc%nominal_value
END SUBROUTINE
```

### 域级编码： `MD_Model` (顶层模型容器域)
#### 设计意图
组合 Node, Element, Material, Section, Step 等所有 L3/L4 容器，形成一个完整的根上下文（类似于 ODB 文件或 INP 的内存表征）。仅有只读属性，交由 `RT_Job` 接管。
