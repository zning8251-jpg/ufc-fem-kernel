# L5_RT 核心运行时层落地推演

> **文档位置**：`docs/05_Project_Planning/PPLAN/12_全栈域级落地矩阵/05_L5_RT_核心运行时层落地推演.md`

## 总体定位
**L5_RT (Runtime & Orchestration)** 是全流程的时序大脑和调度中心。它本身不做任何几何积分或应力更新，而是协调 L3/L4 并将数据喂给 L2 进行代数运算。最后将结果导出给应用层。

---

### 域级编码： `RT_StepDriver` / `RT_Assembly` / `RT_Solver` (已在11专项深度固化)
*(请详见 `11_闭环落地专项` 系列文档。)*
这三大域负责控制非线性迭代时序、多线程装配 CSR、调用外部线性/非线性求解器（NR、L-BFGS、ARC-CW）。

---

### 域级编码： `RT_Job` (作业调度域)

#### 1. 核心意图与完成体目标
- **设计意图**：接管整个 INP 文件转换出来的 `MD_Model`。管理全局级别的初始化、Step 序列轮转、信号捕获（如用户中断 `Ctrl+C` 处理）。
- **完成体目标**：成为 `AP_Facade` (L6) 和底层核心的最后一道隔离带，捕获所有的 `IF_Error` 并确保即使发散也能执行残局清理（写回重启文件）。

#### 2. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `RT_Job_Def.f90` | 作业状态（`JOB_RUNNING`, `JOB_COMPLETED`, `JOB_ABORTED`） |
| _Ctx     | `RT_Job_Ctx.f90` | 全局作业上下文（包含指向 Model、IO Handler的指针） |
| _Algo    | `RT_Job.f90`（旧称 `RT_Job_Algo.f90`）| 调度算法，依次遍历 Model 中的 Steps 执行 `RT_StepDriver_Run` |
| _Brg     | `RT_Job_Brg.f90` | 向外提供 `Job_Submit`, `Job_Abort` 入口 |

#### 3. 底层代码推断
```fortran
SUBROUTINE RT_Job_Submit(job_ctx, err_stat)
    ! 循环处理 Step
    DO i = 1, job_ctx%model%num_steps
        CALL RT_StepDriver_Run(job_ctx%model%steps(i), err_stat)
        IF (IF_Error_IsBad(err_stat)) THEN
            CALL RT_IO_WriteAbortState(...)
            RETURN
        END IF
    END DO
END SUBROUTINE
```

### 域级编码： `RT_IO` 与 `RT_Post` (输入输出与后处理域)

#### 1. 核心意图与完成体目标
- **设计意图**：隔离不同结果格式（如 VTK, ODB, TXT）的写入。计算核心不需要知道现在是在写 VTK 还是别的格式。
- **完成体目标**：由 `RT_StepDriver` 在指定的帧（Frames）触发回调，异步或分块将 `PH_Node_State` 与 `MD_Material_State` 转换为可视化的场数据（FieldData）。

#### 2. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `RT_IO_Def.f90` | IO 模式（`IO_VTK`, `IO_HDF5`） |
| _Ctx     | `RT_IO_Ctx.f90` | 文件句柄管理池，防止频繁 Open/Close |
| _Algo    | `RT_IO.f90`（旧称 `RT_IO_Algo.f90`）| VTK 组装格式化字符串的算法 / HDF5 内存布局转换 |
| _Brg     | `RT_IO_Brg.f90` | `IO_WriteField`, `IO_WriteHistory` |

#### 3. 打通思路与底层代码推断
- **无状态输出**：IO不改变计算状态。传入必须全部为 `INTENT(IN)` 或 `VALUE`。
- IO 写入器通过 `_Brg` 统一暴露，内部 `_Algo` 会将 Voigt 张量展开成 $3 \times 3$ 供可视化软件使用。
