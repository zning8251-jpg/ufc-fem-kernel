# L2_NM 数学与工具层落地推演

> **文档位置**：`docs/05_Project_Planning/PPLAN/12_全栈域级落地矩阵/02_L2_NM_数学工具层落地推演.md`

## 总体定位
**L2_NM (Numerical & Math)** 是纯粹的代数、张量与几何运算引擎。本层依然**没有**物理和力学概念。输入输出只接受纯数值矩阵和向量，不关心其物理含义（是应力还是温度）。这一层是 UFC 执行效率的瓶颈焦点，深度整合 BLAS/LAPACK/MKL 等底层能力。

---

### 域级编码： `NM_Tensor` (张量计算域)

#### 1. 核心意图与完成体目标
- **设计意图**：针对本构模型所需的 2D (ndim=2, nshr=1)、3D (ndim=3, nshr=3) 张量运算（如 Voigt 映射、特征值分解、偏张量提取）进行极致的编译器向量化（SIMD）优化。
- **完成体目标**：提供零内存分配（无隐含 `alloc`）的纯函数计算接口，绝不产生临时内存碎片。

#### 2. 合同卡设计 (Domain Contract)
- **输入依赖**：标量数组、张量阶数标识（1D/2D/3D/Voigt）。
- **输出写回**：计算后的变异张量或标量（如 Mises 等效值）。

#### 3. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `NM_Tensor_Def.f90` | Voigt 下标映射表常数（`IDX_11`, `IDX_12` 等） |
| _Ctx     | `NM_Tensor_Ctx.f90` | 张量计算特有预分配工作区（如特征值计算的临时特征向量空间） |
| _Algo    | `NM_Tensor.f90`（旧称 `NM_Tensor_Algo.f90`）| 张量点积、双点积、偏张量计算、主值计算核心算法 |
| _Brg     | `NM_Tensor_Brg.f90` | 为 L3_MD 提供基于 Voigt 约定的安全入口接口 |

#### 4. 打通思路与底层代码推断
纯函数特征明显，必须标记 `PURE` 或 `ELEMENTAL`：

### 域级编码： `NM_Sparse` (稀疏矩阵结构域)

#### 1. 核心意图与完成体目标
- **设计意图**：封装 CSR（压缩稀疏行）、CSC 甚至 BSR 数据结构，隔离具体的求解器要求。
- **完成体目标**：支持多线程（OpenMP原子加或图着色并发写入）装配，并支持结构自适应扩张。

#### 2. 合同卡设计
- **输入依赖**：元素自由度映射表 (`eq_map`)、局部刚度矩阵 ($K_e$)。
- **输出写回**：将 $K_e$ 无锁拼接入 CSR `values` 数组中对应位置。

#### 3. 十件套物理映射
| 模块类型 | 文件名推断 | 核心职责 |
| -------- | ---------- | -------- |
| _Def     | `NM_Sparse_Def.f90` | 矩阵存储枚举（`CSR`, `CSC`, `DENSE`） |
| _Desc    | `NM_Sparse_Desc.f90`| 稀疏矩阵拓扑结构（`row_ptr`, `col_ind`），一旦符号分解后不可变 |
| _State   | `NM_Sparse_State.f90`| 矩阵的数值载体（`values`）及其并发锁/着色数组 |
| _Algo    | `NM_Sparse.f90`（旧称 `NM_Sparse_Algo.f90`）| 符号装配（Symbolic Factorization）建图算法、快速查询算法 |
| _Brg     | `NM_Sparse_Brg.f90` | `Sparse_AddBlock`、`Sparse_ZeroOut` 门面 |
