# UFC 核心攻坚战：三大硬核任务实施规划 (2026 Phase 6)

## 总则

在底层架构与命名规范（极简三段式）已实现100%收敛的基础上，本阶段将正式切入硬核物理与高性能计算深水区。攻坚顺序严格按照：**解算鲁棒性 -> 内存极速化 -> 物理库多维扩展**的线性路径推进，步步为营。

---

## 阶段一：核心解算逻辑的打磨 (聚焦 `L5_RT` 与 `L2_NM`)

**【目标】** 解决强非线性（如严重软化）下弧长法与牛顿法的发散问题，实现工业级解算鲁棒性。
**【时间/状态】** `Pending` (下一执行步)

- **[ 1.1 迭代策略重构 ]**
  - 在 `RT_Solv_Algo` 中引入混合迭代控制器。实现当残差超过阈值（Divergence Warning）时，拦截致命报错。
  - 开发自动步长切割（Cut-back）与重试机制，即自动将 $\Delta t = \Delta t / 4$，并恢复状态。
- **[ 1.2 弧长法 (Arc-Length) 鲁棒性升级 ]**
  - 梳理 `L-BFGS + ARC-CW` (test_run04) 和 `NR + ARC-CW` (test_run05) 的流转路径。
  - 放宽强软化阶段的弧长正交性判据，增加基于能量范数（Energy Norm）的辅助收敛判定。
- **[ 1.3 状态回滚机制 (State Rollback) 安全性验证 ]**
  - 确保迭代失败退回时，`MD_Mat_State`（如等效塑性应变）等历史变量能被无污染地 100% 还原。

---

## 阶段二：内存体系的优化 (聚焦 `L1_IF` 与 `L4_PH`)

**【目标】** 实现热路径“零分配”，消除内存碎片，提升 Cache 命中率，为 MPI/OpenMP 并行铺路。
**【时间/状态】** `Pending`

- **[ 2.1 物理热路径“零分配”改造 ]**
  - 审查 `PH_Elem_Algo` 和 `PH_Mat_Algo`（特别是单元刚度矩阵与高斯点循环）。
  - 剥离所有动态 `ALLOCATE`，强制改为通过 `IF_Mem_Algo` 从预分配的 Thread-local 工作区借用 Scratch 数组。
- **[ 2.2 数据布局 (AoS vs SoA) 对齐优化 ]**
  - 分析海量单元状态数据在 Fortran 列优先 (Column-major) 下的分布。
  - 调整高频访问数据结构，使得相邻单元或相邻高斯点在物理内存中地址连续。
- **[ 2.3 极致轻量化 Context 穿透 ]**
  - 精简 `RT_Step_Ctx`，全面实施指针别名（Pointer Aliasing）机制，实现 L3 -> L4 热计算时的数据“零拷贝”把手传递。

---

## 阶段三：单元库功能的扩展 (打通 L3/L4/L5)

**【目标】** 落地“单元 / 材料 / 截面”三维正交架构，支撑海量工业级单元族扩展，对齐 Abaqus 规范。
**【时间/状态】** `Pending`

- **[ 3.1 降维流处理 (2D Plane Stress/Strain) 彻底解耦 ]**
  - 改造现有的“2D 嵌入 3D”低效计算流。
  - 强制 2D 连续体单元（CPE4/CPS4 等）在进入 `PH_Mat_Algo` 前，降维至 `ndim=2, nshr=1`（仅3个应力分量）参与真实本构积分计算。
- **[ 3.2 建立单元/材料/截面（Element-Material-Section）正交分派轴 ]**
  - 在 `L5_RT` 装配层建立正交查表法：单元算式逻辑（如 C3D8） $\times$ 积分截面规则（Section） $\times$ 材料本构逻辑（Material）。
  - 切断三者间的硬编码，实现任意组合（例如：壳单元+复合材料铺层截面+各项异性弹塑性本构）。
- **[ 3.3 体积锁死 (Locking) 与沙漏 (Hourglass) 控制 ]**
  - 引入 B-bar / F-bar 法求解塑性不可压缩性导致的锁死。
  - 为一阶减缩积分单元注入独立的沙漏控制刚度评价程序。
- **[ 3.4 微内核提取与代码自动化生成 ]**
  - 提取通用形函数库（Shape Functions）。
  - 编写 Python 生成器脚本，根据配置批量生成剩余 370+ 种变体单元的极简三段式 `.f90` 外壳与装配代理。
