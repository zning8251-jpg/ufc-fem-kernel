## 4. 并行线（不可与「层已打通」混为一谈）

| 并行线 | 门禁/证据 | 与主轴关系 |
|--------|-------------|------------|
| **功能正确性** | 单测 / 集成 / 关键算子金线（如 Phase4 表内 C 列） | **必须通过**；合同齐 ≠ 物理对 |
| **性能 / 热路径** | 热路径零分配策略、Guardian HOT 类规则 | **必须通过**；命名/冗长扫描 **不可替代** |
| **物理 / 回归题** | patch、材料/单元基准题 | **必须通过** |
| **冗长标识符扫描** | `python UFC/tools/scan_verbose_identifiers.py`（默认报告；可选 `--fail-on`） | **卫生线**；不写入「波次 0 已打通」的硬条件，除非团队显式升级 |

---

## 5. 一页结论

- **主轴** = 每条 PR 声明 **层 / 域 / 合同·Bridge·SIO**，并按 **试点 + 执行计划 + Phase4 + `06`/`07`** 落验收。  
- **打通** = **波次 0 → 1 → 2** 上表逐格勾选，而非一次性全库。  
- **并行线** = 正确性 / 性能 / 物理回归 **单独绿**，避免把「洁癖」当「架构完成」。

*维护：波次定义若与 Phase4 / 执行计划冲突，**以执行计划与桥接说明为裁决**；本页仅作叙事汇总与 PR 模板入口。*
