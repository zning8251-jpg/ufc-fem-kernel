- 历史上 **`02` 不是 `01/03/04…` 那样的文件夹**，而是根目录单文件 `02_命名规范_UFC极简三段式草案.md`。  
- 当前 **命名权威** 已在 `UFC/REPORTS/` 与 `UFC_命名与数据结构规范.md`，草案仅作历史对照。  
- 现已建立 **`02_命名与依赖导读/`**：只放导读与历史草案，避免与 REPORTS 双权威。

## 6. 删冗与提炼（执行顺序建议）

1. **断链扫描**：`python UFC/tools/check_docs_health.py UFC/docs --skip-links-from-archive -o UFC/REPORTS/DocsHealth_scan_active-docs.md`（见 `docs/README.md`；报告与待办见 `UFC/REPORTS/`）。  
2. **归类**：根目录规划稿 → `03_实施规划/`；命名相关 → 只保留 `02_导读` + REPORTS。  
3. **合并**：同一主题多版本（v0.1/v1.0）保留最新 + 在旧文顶栏注明「由…取代」。  
4. **删除候选**（需维护者确认后执行）：与 `14–19` **完全重复**的短文本、空壳 README、已被 REPORTS/v5.0 **逐条覆盖**且无历史价值的草稿。  
5. **哲学/心法类 `.txt`/长文**：不挡工程验收；若要瘦身，**移动至** 独立归档目录（见 [`../../DOC_MAINTENANCE_GUIDE.md`](../../DOC_MAINTENANCE_GUIDE.md)）优于硬删。

### 6.1 哲学类 `PPLAN/14–18_*.md`（单独 PR）

与工程入口 **彻底分离** 时：新建 `PPLAN/08_哲学与心法/`（或 `docs/哲学与心法/`），搬迁 `14`–`18` 文件并 **全库替换相对链接**；本步改动面大，**勿与** 断链修复/归档混在同一批合并。搬迁后在 `PPLAN/README.md` 保留一行入口即可。
