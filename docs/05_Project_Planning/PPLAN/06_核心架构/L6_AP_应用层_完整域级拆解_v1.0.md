# L6_AP 应用层 — 完整域级拆解 v1.0

> **层级**: L6_AP (Application Layer)  
> **版本**: v1.0  
> **层级职责**: 输入解析、求解器调用、用户界面、命令行  
> **域级总数**: 5 个  
> **子域总数**: 7 个  
> **功能模块总数**: 17 个  
> **命名前缀**: `AP_` (Application)

---

## 📋 L6_AP 层拓扑结构

```
L6_AP (应用层)
│
├─ Input — INP 文件解析与输入处理
├─ Output — 结果集成与展示
├─ Solver — 主求解器程序入口
├─ Command — 命令行界面
└─ GUI — 图形用户界面 (可选)
```

---

## 🎯 一、五个域级完整拆解

### 1.1 域级 1: **Input** — 输入处理

**职责**: INP 文件解析、模型读取、参数验证

**子域** (2 个):
- **INPParser** — INP 关键字解析
- **Validation** — 输入数据验证

**功能模块** (4 个):
```
AP_Inp_Desc.f90           — 输入配置
AP_Inp_Parser.f90         — INP 文件解析
AP_Inp_Validator.f90      — 数据验证
AP_Inp_Ctx.f90            — 输入上下文
```

### 1.2 域级 2: **Output** — 结果输出

**职责**: 结果集成、显示格式化、多格式导出

**子域** (2 个):
- **ResultFormat** — 结果格式化
- **Export** — 多格式导出 (CSV/VTK/等)

**功能模块** (4 个):
```
AP_Out_Desc.f90           — 输出配置
AP_Out_Format.f90         — 结果格式化
AP_Out_Export.f90         — 多格式导出
AP_Out_Ctx.f90            — 输出上下文
```

### 1.3 域级 3: **Solver** — 主求解程序

**职责**: 程序入口、求解流程控制

**功能模块** (2 个):
```
AP_Solv_Main.f90          — 主程序入口
AP_Solv_Ctx.f90           — 求解器上下文
```

### 1.4 域级 4: **Command** — 命令行界面

**职责**: 命令行参数解析、帮助信息

**功能模块** (3 个):
```
AP_Cmd_Parser.f90         — 参数解析
AP_Cmd_Help.f90           — 帮助信息
AP_Cmd_Ctx.f90            — 命令行上下文
```

### 1.5 域级 5: **GUI** — 图形用户界面 (可选)

**职责**: 前处理、后处理可视化

**功能模块** (4 个):
```
AP_GUI_PreProcess.f90     — 前处理界面
AP_GUI_PostProcess.f90    — 后处理界面
AP_GUI_Render.f90         — 三维渲染 (可选)
AP_GUI_Ctx.f90            — GUI 上下文
```

## 🔗 三、命名规范

**层前缀**: `AP_`

**子域前缀**:
- Input → AP_Inp_*
- Output → AP_Out_*
- Solver → AP_Solv_*
- Command → AP_Cmd_*
- GUI → AP_GUI_*

**特殊约束**:
1. AP_Solv_Main.f90 是程序全局入口，必须调用 L5_RT 的 StepDriver
2. 所有 INPUT 验证必须通过 L3_MD 的约束矩阵
3. OUTPUT 格式必须支持 ODB、CSV、VTK 三种
