## 2️⃣ Coupling维度速查

```
ID | 名称        | 解决方法      | 迭代特性      | 耦合紧密度
───┼────────────┼──────────────┼──────────────┼──────────────
 1 | OneShot    | 单次求解     | 无迭代        | 无耦合
 2 | OneWay     | 单向传递     | 单向迭代      | 单向
 3 | Weak       | 弱耦合迭代   | 松耦合迭代    | 松耦合
 4 | Strong     | 完全迭代     | 紧耦合迭代    | 强耦合
```

**推荐应用**：
```
OneShot  → 单物理场 (结构、热、频域等)
OneWay   → 主从耦合 (热→结构单向传递)
Weak     → 松耦合   (热-结构、流体-热)
Strong   → 紧耦合   (电-结构、流体-结构、多场)
```

## 5️⃣ 编码规则

### 5A. 坐标编码与解码

**编码(转换为Group_ID)**：
```
Group_ID = Solver × 100 + Coupling × 10 + Physics

示例：
[0][2][6] = 0×100 + 2×10 + 6 = 26

对应分析：Standard + Weak + ThermalStruct
(热-结构耦合分析，推荐使用Weak策略)
```

**解码(从Group_ID恢复坐标)**：
```
Group_ID = 26
  ↓
Solver = 26 / 100 = 0
Coupling = (26 % 100) / 10 = 2
Physics = 26 % 10 = 6
  ↓
[0][2][6] ✓
```

## 7️⃣ Geo(岩土)分析特殊处理

### 编码方式

**岩土分析的UFC编码**：
```
[Solver][Coupling][0] + Material_Family=GEO

示例：
[0][0][0] + Material=Mohr-Coulomb soil
  ↓
自动识别为岩土准静态分析
```

### 实现特性

```
L3_MD层：
  标记 has_geo_material_family = .TRUE.
  
L4_PH层：
  IF (Material_Family == MAT_GEO) THEN
    CALL enable_geo_processor()
    CALL setup_initial_stress_balance()
  END IF
  
L5_RT层：
  初始应力平衡 → 工况叠加 → 标准求解
```

## 8️⃣ 多求解器耦合(FSI)特殊处理

### FluidStruct [4][3][8]

```
编码：[4][3][8]

含义：CFD求解器 + 强耦合 + 流体-结构双场

自动行为：
  1. L4_PH识别Physics=8
  2. 自动启用SOLVER_STANDARD作为辅助
  3. 从Interaction对象获取接触面定义
  4. L5_RT Harness启用FSI耦合循环

耦合算法：
  DO iter = 1, n_strong_iterations
    流体求解 (CFD)
    结构求解 (Standard)
    网格变形 (Mesh Update)
    检查收敛
  END DO
```

### FluidThermal [4][2][9]

```
编码：[4][2][9]

含义：CFD求解器 + 弱耦合 + 流体-热双场

算法：
  温度场从CFD获得 →
  传递到热求解器 →
  求解结构温度分布 →
  反传边界条件 →
  下一时间步
```

## 9️⃣ 日常检查清单

### 新增分析类型时

- [ ] 确定主物理场 → Physics维度
- [ ] 确定求解算法 → Coupling维度
- [ ] 确定求解器 → Solver维度
- [ ] 查约束矩阵 → 是否合法？
- [ ] 确定材料族 → 是否需要特殊标记(如Geo)？
- [ ] 确定多求解器需求 → 是否需要Harness编排？
- [ ] 更新PROC映射表
- [ ] 生成单元测试用例

- [ ] L3_MD: 更新Group_DESC结构体
- [ ] L4_PH: 更新约束矩阵与路由逻辑
