## 二、瘦身策略

### 2.1 策略1: Command域合并 (推荐) ⭐

**方案**: 将50个Command文件合并为5-8个族级文件

| 合并后文件 | 包含命令数 | 说明 |
|-----------|----------|------|
| AP_Inp_Cmd_Node.f90 | 8 | 节点相关 (*Node, *Nset) |
| AP_Inp_Cmd_Element.f90 | 12 | 单元相关 (*Element, *Elset) |
| AP_Inp_Cmd_Material.f90 | 10 | 材料相关 (*Material, *SolidSection) |
| AP_Inp_Cmd_Step.f90 | 8 | 步相关 (*Step, *Boundary) |
| AP_Inp_Cmd_Load.f90 | 7 | 载荷相关 (*Cload, *Dload) |
| AP_Inp_Cmd_Output.f90 | 5 | 输出相关 (*Output, *History) |

**预期效果**: 50→6个文件 (-44个文件, -88%)

### 2.2 策略2: 删除低优先级命令

**方案**: 保留核心30个命令，删除20个低优先级/ deprecated命令

**保留**:
- *Node, *Element, *Nset, *Elset
- *Material, *SolidSection, *Elastic, *Plastic
- *Step, *Static, *Boundary, *Cload, *Dload
- *Output, *History, *Field
- *Assembly, *Instance, *Part

**删除** (移至Input/Command/Deprecated/):
- *Surface (可用*Nset替代)
- *Contact Pair (可用*Contact替代)
- *Amplitude (可用表格替代)
- 其他20个低频命令

**预期效果**: 50→30个文件 (-20个文件, -40%)

### 2.3 策略3: 域级合并

**方案**: 将L6_AP层8个域合并为4个域

| 合并后域 | 原域 | 文件数 |
|---------|------|--------|
| **Input** | Input (75) | 75 |
| **Output** | Output (6) + Job (4) | 10 |
| **Config** | Config (2) + Registry (1) + Solver (1) | 4 |
| **Bridge** | Bridge (4) + UI (7) | 11 |

**预期效果**: 8域→4域，文件数不变但结构更清晰
