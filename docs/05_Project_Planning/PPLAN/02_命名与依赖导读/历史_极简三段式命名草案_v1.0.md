# UFC 极简三段式命名规范 (v1.0 草案)

> **状态（2026-04-25）**：**历史对照稿**。当前命名与后缀策略以 `UFC/REPORTS/UFC_统一命名方案_层级域级功能三级体系_v1.0.md`、`UFC/REPORTS/UFC_命名规范与接口标准_v2.0.md` 及 `UFC/docs/UFC_命名与数据结构规范.md` 为准。详见同目录 `[README.md](README.md)`。

## 1. 核心思想

采用 **“前缀_中段_后缀”** 的三段式结构，兼顾跨层调用的命名空间区分度和 Fortran 代码行宽限制。

- **文件名严格等于 Module 名**，做到所见即所得。
- **公共接口必须携带模块前缀**，避免全局命名冲突。
- **私有接口及局部变量极致精简**，降低视觉负担。
- **特例排除**：`L2_NM/ExternalLibs` 等第三方外部库保持其原生结构，不参与此命名重构。

---

## 2. 三段式结构定义

### 2.1 前缀 Prefix（层级标识，2个大写字母）

彻底废弃 `L1`_、`UF_` 等冗长前缀。

- `IF` = L1_IF 基础设施层
- `NM` = L2_NM 数值层
- `MD` = L3_MD 模型层
- `PH` = L4_PH 物理层
- `RT` = L5_RT 运行时层
- `AP` = L6_AP 应用层

### 2.2 中段 Domain（域/子域简写，PascalCase 大驼峰）

由 1~~2 个标准缩写组合而成，控制在 2~~3 个音节。

**【通用域级字典】**

- 单元：`Element` -> `Elem`
- 材料：`Material` -> `Mat`
- 网格：`Mesh` -> `Mesh`
- 载荷/边界：`LoadBC` -> `LBC`
- 接触/相互作用：`Contact` -> `Cont`, `Interaction` -> `Int`
- 内存：`Memory` -> `Mem`
- 矩阵/求解器：`Matrix` -> `Mtx`, `Solver` -> `Solv`
- 运行时控制：`StepDriver` -> `Step`, `Assembly` -> `Asm`
- 输出/回写：`Output` -> `Out`, `WriteBack` -> `WB`
- 并行/多线程：`Parallel` -> `Par`

**【常见子域字典】**

- 实体单元：`Solid2D` -> `Sld2D`, `Solid3D` -> `Sld3D`, 壳`Shell`, 梁`Beam`
- 塑性/弹性：`Plastic` -> `Plast`, `Elastic` -> `Elas`, `HyperElastic` -> `Hyper`
- 损伤/蠕变：`Damage` -> `Dam`, `Creep` -> `Creep`

*示例组合：* `ElemSld3D`, `MatPlast`, `MtxSolv`

### 2.3 后缀 Suffix（实体属性，1个单词缩写）

表示该文件的具体功能角色（契合“四型”架构）。

- `Def`：定义层（存放常数、基础 Derived Types 的声明，通常不含复杂方法）
- `Ctx`：上下文/状态层（State/Context/Descriptor 的实例包装或管理逻辑）
- `Algo`：算法层（核心计算子程序、积分、求解等）
- `Brg`：桥接层（跨层通信的门面接口 Face/Bridge）

## 3. 全局命名与变量示例清单

为了验证该规范能否满足日常开发，以下列举所有层级的代码元素示例。

### 3.1 文件与 Module 级命名

严格遵循 `[Prefix]_[Domain]_[Suffix]`。

- `PH_ElemSld3D_Algo.f90` (定义 `MODULE PH_ElemSld3D_Algo`)
- `MD_MatPlast_Def.f90` (定义 `MODULE MD_MatPlast_Def`)
- `RT_Step_Ctx.f90` (定义 `MODULE RT_Step_Ctx`)
- `IF_Mem_Brg.f90` (定义 `MODULE IF_Mem_Brg`)

### 3.2 派生类型 (Derived Types) 命名

属于 `PUBLIC` 暴露数据，命名格式：`[Prefix]_[Domain]_[类别]`

- **配置描述 (Desc)**：`MD_MatPlast_Desc` (材料参数，不随时间变化)
- **演化状态 (State)**：`PH_ElemSld3D_State` (应力、应变历史，随积分点变化)
- **运行时上下文 (Ctx)**：`RT_Global_Ctx` (全局上下文把手)

### 3.3 子程序与函数 (Subroutines & Functions)

- **PUBLIC 对外接口**：`[Prefix]_[Domain]_[动词简写][宾语]`
  - `CALL PH_ElemSld3D_CalcStiff(...)`  (物理层：计算实体刚度)
  - `CALL MD_MatPlast_InitDesc(...)`  (模型层：初始化塑性参数)
  - `CALL RT_Step_Advance(...)`       (运行时：分析步推进)
  - `CALL NM_Mtx_SolveSystem(...)`    (数值层：求解矩阵系统)
- **PRIVATE 内部私有实现**：`[动词]_[宾语]` (snake_case，完全抛弃前缀)
  - `CALL invert_jacobian(...)`
  - `CALL check_yield_surface(...)`
  - `CALL allocate_buffer(...)`

### 3.4 内部变量约定 (Variables & Constants)

- **全局参数/常数 (Constants)**：全大写，带前缀 `C`_ 或 `P_`。
  - `C_PI`, `C_ZERO`, `C_TOLERANCE`
- **实例对象形参 (Dummy Arguments)**：极简，固定词汇。
  - `desc` (类型为 Type(..._Desc))
  - `state` (类型为 Type(..._State))
  - `ctx` (类型为 Type(..._Ctx))
- **局部变量 (Local Variables)**：下划线 `snake_case`。
  - `num_nodes`, `curr_stress`, `dt`, `elem_id`

## 4. 综合代码视觉效果展示 (Before / After 对比)

**【重构前：冗长而难以阅读的旧版 Fortran】**

```fortran
MODULE UF_L4_PH_Element_Solid3D_Algorithm
    USE UF_L3_MD_Element_Solid3D_Types
    IMPLICIT NONE
    PUBLIC :: UF_L4_PH_Element_Solid3D_Stiffness_Evaluate
CONTAINS
    SUBROUTINE UF_L4_PH_Element_Solid3D_Stiffness_Evaluate(UF_Global_Ctx, UF_Solid3D_Descriptor, UF_Solid3D_State, Element_Stiffness)
        TYPE(UF_Global_Runtime_Context), INTENT(IN) :: UF_Global_Ctx
        TYPE(UF_L3_MD_Element_Solid3D_Desc), INTENT(IN) :: UF_Solid3D_Descriptor
        TYPE(UF_L3_MD_Element_Solid3D_State), INTENT(INOUT) :: UF_Solid3D_State
        REAL(8), INTENT(OUT) :: Element_Stiffness(24, 24)
        
        INTEGER :: Number_Of_Integration_Points
        ...
        CALL UF_L4_PH_Element_Solid3D_Invert_Jacobian(UF_Solid3D_State%Jacobian)
    END SUBROUTINE
END MODULE UF_L4_PH_Element_Solid3D_Algorithm
```

**【重构后：采用极简三段式的工业级 Fortran】**

```fortran
MODULE PH_ElemSld3D_Algo
    USE MD_ElemSld3D_Def
    USE RT_Global_Def
    
    IMPLICIT NONE
    PRIVATE
    
    ! 仅暴露必要对外的计算接口
    PUBLIC :: PH_ElemSld3D_CalcStiff

CONTAINS

    ! [公共接口] 评估实体单元刚度
    SUBROUTINE PH_ElemSld3D_CalcStiff(ctx, desc, state, ke)
        TYPE(RT_Global_Ctx),      INTENT(IN)    :: ctx
        TYPE(MD_ElemSld3D_Desc),  INTENT(IN)    :: desc
        TYPE(MD_ElemSld3D_State), INTENT(INOUT) :: state
        REAL(8),                  INTENT(OUT)   :: ke(:,:)
        
        INTEGER :: num_ipts
        ...
        ! 调用私有函数，抛弃前缀
        CALL invert_jacobian(state%jac)
        ...
    END SUBROUTINE
    
    ! [私有实现] 雅可比矩阵求逆
    SUBROUTINE invert_jacobian(jac)
        REAL(8), INTENT(INOUT) :: jac(3,3)
        ...
    END SUBROUTINE

END MODULE PH_ElemSld3D_Algo
```
