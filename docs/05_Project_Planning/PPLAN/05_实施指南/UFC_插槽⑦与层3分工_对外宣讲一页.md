## 一句话

- **推理**：①–⑥ 插槽，FEM **调用**已训练 AI（**层 1**，可 **SKU-A** 分阶段交付）。  
- **训练 / 优化闭环**：**⑦** 管 **「矩阵与伴随方程怎么解」**；**层 3** 管 **「物理残差对设计变量怎么求导 ∂R/∂θ」**——**缺一不可谈「严格物理梯度」**，但**里程碑可拆分**。

---

## ⑦ 与层 3：各管什么？

| 维度 | **插槽⑦ `AI_AdjointSolver`（层 2）** | **可微分物理引擎（层 3）** |
|------|----------------------------------------|----------------------------|
| **归属层** | **L2_NM**（数值线性代数、稀疏结构） | **L4_PH / Element**（本构、单元、物理语义） |
| **解决什么问题** | 给定 **K**（或等价算子）与 **∂J/∂u**，解 **Kᵀ·λ = ∂J/∂u**；组织 **TransposeSolve**、与正向分解复用 | 给定 **θ**，计算 **∂R/∂θ**（残差对设计/材料/几何等参数的灵敏度），**物理一致** |
| **有没有「物理公式」** | **无**——不推导本构、不组装单元物理项 | **有**——导数来自控制方程离散与材料模型 |
| **何时运行** | **仅**离线优化 / 训练场景；**禁止**进常规增量步热路径 | 同上，随层 2/3 能力标志 **按需**启用 |
| **和 AI 的关系** | 把 **FEM 端梯度**送到外层，使 **Loss→θ** 链可闭合 | 提供 **∂R/∂θ**，使 **离散伴随**公式 **∂J/∂θ = −λᵀ·(∂R/∂θ)** 可用 |

---

## 公式链（便于背板）

```
正向（热路径，既有 CAE）：   R(u; θ) = 0  →  Newton / 线性化
伴随（冷路径，训练侧）：     Kᵀ · λ = ∂J/∂u          ← ⑦ 主战场（求解器侧）
灵敏度（冷路径，物理侧）：   ∂J/∂θ = − λᵀ · (∂R/∂θ)   ← ∂R/∂θ 由层 3 提供
```

- **只有 ⑦、没有 ∂R/∂θ**：缺少 **物理 Jacobian 块**，一般**无法**在工程上闭合「严格离散伴随」灵敏度。  
- **只有 ∂R/∂θ、没有 ⑦**：有了局部导数，仍要解决 **大规模伴随方程**与 **Kᵀ 与数据流**——⑦ 正是这一环的**正式插槽化**落点。

## 与「六插槽推理」的关系

| 侧 | 插槽 | 数据方向 | 说明 |
|----|------|----------|------|
| **推理** | ①–⑥ | FEM → AI（前向调用模型） | 与训练侧 **正交**；可单独里程碑交付（**SKU-A**） |
| **训练** | **⑦** + **层 3** | AI ← FEM（梯度回传） | **SKU-B 完整闭环** 的必要能力（可分期实现） |

## 对外口径（可选读）

1. **UFC 核心仍是确定性 Fortran CAE 内核**；AI 通过 **合同化插槽**接入，**默认热路径不变**。  
2. **⑦ 不是「又一个神经网络」**，而是 **伴随 / 转置求解**在架构上的**命名插槽**；**层 3 不是第二个 ORT**，而是 **物理残差对 θ 的导数**落在 **单元/材料**层。  
3. **梯度实现节奏**（FD → 手写关键路径 → 局部 AD 等）见集成规范 **§1.2.2**，**不**把「全场自动微分」写进 P0/P1 默认承诺（总纲 **§0.4**）。
