# 基于模板的存量资产渐进式重构方案 (绞杀者模式实施路线)

## 1. 核心理念：拒绝休克疗法，采用渐进绞杀
在面对拥有历史包袱的底层有限元内核时，直接修改核心文件（“休克疗法”）往往导致牵一发而动全身，引发无法定位的编译或计算错误。
本方案采用业界最佳实践——**“绞杀者无花果模式 (Strangler Fig Pattern)”** 结合 **“样板间模式”**，通过具体问题具体分析、先建模板后适配的流水线方式，安全、平稳地消化历史技术债。

## 2. 实施战术：四步走流水线 (The 4-Step Assembly Line)

### 步骤一：打造纯血统的“标准样板间” (Build the Golden Template)
*   **动作**：在不动任何旧代码的前提下，在对应目录（如 `L3_MD/Material/Template`）中建立纯粹的“样板间”模块。
*   **标准**：
    *   严格应用 17 件套架构（分为 `_Def`, `_Ext`, `_Core`, `_Proc` 等）。
    *   严格应用多级辅 TYPE 嵌套规范。
    *   严格采用五参/六参标准 IO，并在注释中标明 `[IN]` / `[OUT]`。
    *   严格套用《方程到代码映射手册》的数学骨架。
*   **目标**：以此模板为标尺，任何旧代码都要削足适履地“套进”这个模具。

### 步骤二：存量资产“剥壳抽芯”大分类 (Asset Triage & Core Extraction)
*   **动作**：对旧有资产（如散落的 `UF_Mat_Elast_xxx.f90`, `Mises_Plasticity.f90`）进行外科手术式的解剖。
*   **分类**：
    1.  **配置/解析外壳代码**：全部抛弃或重构为 `_Populate` 时相。
    2.  **数据流转/中间变量代码**：全部抛弃，用新的 `_Ctx` 辅 TYPE 替代。
    3.  **纯粹物理数学内核**（如应力偏张量计算、径向返回、刚度阵组装）：这是我们要提取的**“核芯资产”**。这些资产往往是纯数学公式，不依赖全局系统，必须原封不动地保留。

### 步骤三：套壳适配与桥接 (Adapter Pattern Wrapping)
*   **动作**：将提取出的“核芯资产”（纯数学代码）放入“标准样板间”的 `_Core_Eval` 时相中。
*   **适配**：在 `_Core_Eval` 子程序的最上方，利用一行行代码将优雅的辅 TYPE 数据解包为数学内核所需的纯量或小矩阵，算完后再打包回去。
    *   *旧代码改造即成为了简单的变量映射填空题。*

### 步骤四：影子并行与绞杀替换 (Shadow Run & Strangler Deprecation)
*   **动作**：新旧系统并存一段时间。
*   **验证**：利用第 13 件 Test 测试规范，构造一个单单元测试。将相同的应变增量同时喂给旧子程序和新模板，断言其应力与刚度输出是否达到 $10^{-12}$ 精度的完全一致。
*   **绞杀**：测试通过后，将原旧文件打上 `DEPRECATED` 标签并废弃其路由，在 Registry（第 10 件路由组件）中将控制权彻底切向新模板模块。

## 3. P1 切片 (L3_MD_Material) 实施清单
1. 建立 `MD_Mat_Standard_Template.f90` (包含 Elastic 与 Mises 样板)。
2. 解剖现存的材料代码，分类梳理出其参数输入与状态更新逻辑。
3. 改造 `MD_Mat_Def.f90`，引入辅 TYPE，支撑样板间的运行。
4. 跑通单元极简回归测试，正式废除旧材料算法。
