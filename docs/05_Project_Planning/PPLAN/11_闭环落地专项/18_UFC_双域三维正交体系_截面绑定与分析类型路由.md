## 1. 物理-几何路由：【单元 - 截面 - 材料】三维正交与兼容矩阵

您指出的“不是所有材料都适配所有单元”，切中了 L3_MD 与 L4_PH 之间最核心的“阻抗匹配”问题。`Section`（截面）绝不是一个简单的厚度数字，它是衔接数学插值（Element）与物理本构（Material）的**强制降维协议（Kinematic Mapping）**。

### 1.1 三维架构定义
*   **X轴（几何插值 - Element）**：决定节点拓扑、形函数、积分点分布。如：CPS4（二维四节点）、C3D8（三维八节点）、S4R（四节点壳）。
*   **Y轴（运动学截面 - Section）**：决定应力/应变空间的降维投影与截面积分。如：`SolidSection` (实体截面), `ShellSection` (壳截面), `BeamSection` (梁截面)。
*   **Z轴（本构响应 - Material）**：决定应力-应变关系的物理法则。如：线弹性、各向同性塑性、超弹性。

### 1.2 兼容性验证红线 (Compatibility Matrix)
在 `Populate` 阶段初始化模型时，必须由 `Section` 发起严格的阻抗匹配检查。如果乱连，直接抛出异常（这也是防范维度坍缩的最佳防线）：

| 单元类型 (L4) | 截面类型 (桥梁) | 材料支持的空间要求 (L3) | 兼容性判定 | 错误示例拦截 |
| :--- | :--- | :--- | :--- | :--- |
| **C3D8** (3D实体) | `SolidSection` | 必须支持 `3D_Stress` (6分量) | **✅ 完美匹配** | 禁连 1D 梁材料 |
| **CPS4** (2D平面应力)| `SolidSection` | 必须支持 `Plane_Stress` (3分量) | **✅ 降维匹配** | 禁连 不可压缩超弹性材料（需厚度向应变解耦） |
| **S4R** (3D壳) | `ShellSection` | 必须支持 `Plane_Stress`，且需定义厚度积分点 | **✅ 厚度积分匹配** | 禁连 纯 3D 块体本构 |
| **T2D2** (2D桁架) | `TrussSection` | 必须支持 `1D_Stress` (1分量) | **✅ 1D投影匹配** | 禁连 复杂的 3D 弹塑性本构 |

**结论**：`Section` 是强类型的检查站，它保证了传给 L3_MD 的 `Ctx` 中的 `ndim` 和 `nshr` 绝对安全合法。

## 2. 分析步驱动路由：【物理场 - 求解器 - 耦合形式】三维正交设计

您回忆起的第二个三维正交设计，是整个 `L5_RT`（求解运行域）的最高指挥棒。它定义了一个 `Step`（分析步）究竟该怎么往下走。

这也是区分“玩具代码”与“工业巨兽”的分水岭。一个真正的通用软件，不可能为“热力耦合”写一套代码，为“流固耦合”写一套代码，而是通过这三个维度的选项**正交组合（Orthogonal Assembly）**。

### 2.1 宏观指挥棒的三维定义
*   **维度 A：物理场 (Physics)**
    决定需要激活哪些自由度（DOF）和哪些物理残差方程。
    *   `Structural` (结构力学：位移 DOF)
    *   `Thermal` (传热学：温度 DOF)
    *   `Porous` (渗流力学：孔隙水压 DOF)
    *   `Acoustic` (声学：声压 DOF)
*   **维度 B：求解算子 (Solver Engine)**
    决定线性代数或时间积分的推进引擎（不管物理公式是什么，只管怎么解矩阵）。
    *   `Static_Implicit` (静力隐式：常规 Newton-Raphson)
    *   `Dynamic_Implicit` (动力隐式：HHT-$\alpha$ 或 Newmark-$\beta$)
    *   `Dynamic_Explicit` (动力显式：中心差分法，无需拼矩阵)
    *   `Arc_Length` (弧长法 / Riks 法：追踪极值点软化)
    *   `Eigen_Lanczos` (特征值求解：模态分析)
*   **维度 C：耦合形式 (Coupling Strategy)**
    当存在多个物理场时，决定偏微分方程组的求解策略。
    *   `Uncoupled` (无耦合/单一场)
    *   `Fully_Coupled` (强耦合/整体矩阵：如热力全耦合，组装非对称巨型雅可比矩阵)
    *   `Sequentially_Coupled` (弱耦合/交错求解：先解热得到温度，温度作为载荷传给力学解位移)

### 2.2 路由组合的工业级威力 (Routing Matrix)

当用户在脚本里定义一个分析步时，他实际上是在这三个轴上拨动开关。UFC 底层将根据这三个开关，自动装配计算流程：

*   **组合 1**：`[Structural] + [Arc_Length] + [Uncoupled]`
    👉 触发 **结构屈曲软化追踪**（您提到的 `test_run04/05` 的强非线性收敛场景）。L5_RT 会挂载弧长法求解器，剥离负载因子。
*   **组合 2**：`[Structural & Thermal] + [Static_Implicit] + [Fully_Coupled]`
    👉 触发 **热机强耦合分析**。此时 L4_PH 在 `Assemble` 时，必须同时组装力学残差、热学残差，以及交叉耦合刚度矩阵 $\frac{\partial F}{\partial T}$ 和 $\frac{\partial Q}{\partial U}$。
*   **组合 3**：`[Structural] + [Dynamic_Explicit] + [Uncoupled]`
    👉 触发 **显式动力学碰撞分析**。此时 L5_RT 的控制流将彻底绕过全局大矩阵组装（跳过 `GlbItrAsm`），直接走集中质量阵的局部对角线求解！
