## 1. 「完备」判据（合同侧）

下列各项须在对应域的 **`CONTRACT.md` 中可核对**（允许合并小节，但审查时能**逐条指出段落或表格**）。与 `06` 验收表 **A4 / A7–A11** 一一对应。

| 代号 | 含义 | 通过标准（摘要） |
|------|------|------------------|
| **A4·四链** | 四链占位与落点 | 理论 / 逻辑 / 计算 / 数据 **各 ≥1 句**本域可核对，且各能 **指到** TYPE、过程、`*_Proc` 或数据流片段 |
| **A4·因果** | 因果说明（推荐） | 2–3 句自然语言或 trace 级 `(trigger; upstream)`；**禁止**「第五链」合同用语（见 `UFC/docs/02_Developer_Guide/UFC_数据结构与结构体规范.md` §5.6） |
| **A7** | 层与域坐标 | **Layer + Domain 路径 + 功能集/子域** 与 `ufc_core/**` 目录一致 |
| **A8** | 内外边界与 Bridge | 四型对内职责；对外 API；跨层仅 **T / B / S** 闭集（与 A6 域际子表一致） |
| **A9** | RT 六参（若适用） | 写明五参/六参及 `RT_Com_Base_Ctx` 与域 `_Ctx` 正交；**不适用须 N/A + 一句理由**（典型：纯 L3 冷数据域） |
| **A10** | 横切契约与可观测 | 版本/序列化/稳定字段（若适用）；观测与 `chain_hint` 仍落四链闭集（若适用）；不适用 **N/A** |
| **A11** | 验证与 Harness | 可执行 **测试命令** + **`run_harness.py` / `uhc.py` 等**清单，或与 **C1–C4** 显式互链且仍能落到具体命令 |

**单行勾选含义**：下列 `- [ ] \`Lx_…/域\`` 表示「该域 `CONTRACT.md` 已满足上表 **A4 + A7–A11**（含允许的 **N/A** 声明）」。

**维护**：若在 L3/L4/L5 **新增**带 `CONTRACT.md` 的域，须在本文件对应层 **追加一行** `- [ ]`，并 bump 本文档「域计数」脚注。
