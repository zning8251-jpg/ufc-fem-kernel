## 1. 维度无冗余证明：解开时、空、动的纠缠

在 UFC 三维正交体系中，任何一个子程序模块的职责，都可以用一个三维向量 $(X, Y, Z)$ 完美定义。三者互不干涉，完全正交：

### X 轴：空间作用域 (Space Scope - "Where")
*完备性证明：有限元本质是对连续介质的离散。只要是离散，就逃不出以下四个绝对物理空间尺度，绝无遗漏。*
1.  **`Global` (全场/系统级)**：对应全局大矩阵 $\hat{K}$ 和总残差向量 $\hat{R}$。
2.  **`Region` / `Surf` (区域/面级)**：对应接触面（Contact Pair）、特定边界集（Node Set / Element Set）。
3.  **`Elem` (单元/网格级)**：对应单个单元的局部矩阵 $K_e$。
4.  **`Node` / `Pt` (节点/积分点级)**：对应最底层的应力应变 $\sigma, \varepsilon$ 或节点位移 $u$。

### Y 轴：时相嵌套树 (Time Phase - "When")
*完备性证明：有限元求解是一个四重嵌套的 `while` 循环（Job $\to$ Step $\to$ Inc $\to$ Iter）。任何一个循环，在计算机科学的状态机模型中，必定且仅包含五个生命周期切面。两者正交组合，覆灭所有时序漏洞。*
*   **循环层级 (Loop Level)**：`Job` (任务) $\to$ `Step` (分析步) $\to$ `Inc` (增量步) $\to$ `Iter` (牛顿迭代)。
*   **生命切面 (Life Slice)**：
    *   `Init`：循环开始前的初始化。
    *   `Predict` / `Trial`：求解前的预估（例如显式积分的试探步）。
    *   `Commit`：循环成功收敛，状态正式固化（$t_{n+1} \to t_n$）。
    *   `Rollback`：循环发散（如非线性不收敛），状态回滚丢弃。
    *   `Finalize`：循环彻底结束，销毁释放。

### Z 轴：核心动作 (Core Action - "What")
*完备性证明：从数据流（Data Flow）的图灵机视角来看，所有的数值算法只做三件事：读取（Fetch）、计算（Compute）、写回（Store）。FEM 动作可精准映射为这五大原语。*
1.  **`Populate` (数据灌注)**：从全局/上一层抓取数据，装配成本地的 `Ctx` 或 `Desc`（读）。
2.  **`Evaluate` (测算/核心计算)**：纯粹的数学物理公式计算，产生 `Args` 输出（算）。
3.  **`Assemble` (组装/散射)**：将局部的刚度/残差或节点力，累加拼接到上一层的全局结构中（写）。
4.  **`Map` / `Extrapolate` (场映射)**：空间层级间的数据投影（如高斯点应力外推到节点）（算+写）。
5.  **`Export` (输出)**：将结果写入到 ODB 数据库或日志（终极写）。

## 2. 完备性检验：刻画所有 FEM 极限场景

如果这个三维架构是完备的，那么有限元中任何晦涩、复杂的物理或数值过程，都能在这个三维坐标系中找到唯一的精准落点，并生成毫无歧义的签名（`[Domain]_[Feature]_Proc_[X][Y][Z]`）。

让我们用四个最复杂的 FEM 场景来进行“极限压力测试”：

### 场景 A：材料的弹塑性返回映射（Return Mapping）
*   **物理描述**：在牛顿迭代中，积分点上的应力超出了屈服面，需要进行塑性修正计算。
*   **三维定标**：
    *   $X$ (空)：`Pt` (积分点级)
    *   $Y$ (时)：`Iter_Run` (迭代步内)
    *   $Z$ (动)：`Evaluate` (核心物理计算)
*   **UFC 签名**：`MD_Mat_Proc_PtIterEvaluate`
*   **完美匹配**：只传该点的 `Ctx`，输出该点的应力和一致切线刚度，无状态污染。

### 场景 B：单元刚度矩阵组装
*   **物理描述**：单元计算完积分点后，乘以形函数和雅可比，组装成 $10 \times 10$ 的局部矩阵，并叠加到全局大矩阵中。
*   **三维定标**：
    *   $X$ (空)：`Elem` (单元级) $\to$ 影响到 `Global`
    *   $Y$ (时)：`Iter_Run` (迭代步内)
    *   $Z$ (动)：`Assemble` (散射写回)
*   **UFC 签名**：`PH_Elem_Proc_ElemIterAssemble`
*   **完美匹配**：动作极度清晰，这里绝对不可能发生 `Commit` 固化行为。

### 场景 C：增量步不收敛导致的时间步切割（Cutback）
*   **物理描述**：牛顿迭代超出了最大次数依然发散，求解器决定丢弃当前增量步，把时间步长 $\Delta t$ 缩小一半重算。
*   **三维定标**：
    *   $X$ (空)：`Global` (全系统)
    *   $Y$ (时)：`Inc_Rollback` (增量步发散回滚)
    *   $Z$ (动)：`Evaluate` (重新计算步长) / `Populate` (重置状态)
*   **UFC 签名**：`RT_Solv_Proc_GlobalIncRollback`
*   **完美匹配**：彻底阻断了脏数据污染。所有 `State` 将在此刻执行回滚，回到 $t_n$。

### 场景 D：高斯点应力外推至节点用于云图显示
*   **物理描述**：当前增量步完全收敛后，需要把高斯点上的应力，通过形函数逆矩阵外推到节点上，以便于画彩色云图。
*   **三维定标**：
    *   $X$ (空)：`Elem` 到 `Node` (跨尺度投影)
    *   $Y$ (时)：`Inc_Commit` (增量步收敛后)
    *   $Z$ (动)：`Map` (场映射)
*   **UFC 签名**：`PH_Elem_Proc_ElemIncMap`
*   **完美匹配**：清晰定义了这只是一个后处理步骤，绝不会在 `Iter` 循环中浪费算力。
