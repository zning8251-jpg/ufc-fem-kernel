# 24_UFC_大规模求解器布阵_AMG与外挂数学库战略

## 1. 概念澄清：多重网格法（MG）在 FEM 中的宿命

您对计算流体力学（CFD）和结构有限元（FEM）底层求解逻辑的边界，把握得极其精准！

### 1.1 几何多重网格法（GMG - Geometric Multigrid）
*   **来源与应用**：发源于 CFD 和有限差分法（FDM），它需要真正的“物理网格”。算法在密网格上消除高频误差，然后将残差投影到物理上更粗的网格上消除低频误差。
*   **FEM 的灾难**：在处理复杂的 3D 工业 CAD 模型时（如带圆角、孔洞的不规则四面体/六面体混合网格），让算法自动生成一套“物理粗网格”是极其困难甚至不可能的。因此，**GMG 基本上被通用结构 FEM 工业软件所抛弃**。

### 1.2 代数多重网格法（AMG - Algebraic Multigrid）
*   **FEM 的救星**：您提到的基于“代数聚类”的正是 AMG！它**完全不需要知道物理网格长什么样**。它纯粹依靠刚度矩阵 $K$ 的图论拓扑结构（非零元权重，强耦合与弱耦合连接），在代数层面自动将变量“聚类（Coarsening）”，生成虚拟的粗层方程。
*   **地位**：AMG 是目前人类已知的、求解千万级至十亿级自由度非结构化 FEM 线性方程组**最强大、扩展性最好的算法（通常作为 PCG/GMRES 的预处理子 Preconditioner）**。

## 2. 核心决策：我们需要自己开发 AMG 吗？

**结论是极其坚决的：绝对不要自己开发！**

### 2.1 为什么不自研 AMG？
*   **数学复杂度极高**：AMG 的代数粗化策略（如 Ruge-Stüben 粗化、聚合粗化 Smoothed Aggregation）和插值算子设计，属于纯粹的计算数学家领域。一个工业级的 AMG 求解器（如 LLNL 开发的 Hypre）背后是几十个博士几十年的心血。
*   **并行难度极大**：在 MPI 分布式环境下，AMG 的网格建立阶段（Setup Phase）需要极其复杂的图划分和跨节点通信，普通团队自己写极易导致死锁或性能灾难。
*   **不符合 UFC 哲学**：UFC 的核心竞争力在 **L3 (物理本构)**、**L4 (单元技术)** 和 **L5 (非线性演化架构)**。线性方程组求解属于最底层的 **L2_NM (数值数学层)**。我们应该做的是**“集成与驭使”**，而不是“重新发明轮子”。

## 3. UFC 的 L2_NM 求解器大布阵战略

我们在 `L2_NM` 层的设计，必须采用**“门面模式（Facade）+ 桥接防腐层（Bridge）”**，将外部顶级数学库封装为 UFC 的标准化黑盒接口。

### 3.1 直接法（Direct Solvers）—— 稳健之王
*   **应用场景**：自由度 $< 500$ 万，或者高度病态、不对称（如接触、大变形、拉格朗日乘子引入0主元）的矩阵。
*   **外挂核武**：
    *   **Intel MKL PARDISO**：单机共享内存（OpenMP）下的无敌王者。
    *   **MUMPS / SuperLU_DIST**：跨节点（MPI）的首选。
*   **UFC 调用策略**：不管多非线性，只要内存塞得下，优先选直接法，它能保证牛顿迭代方向最准。

### 3.2 迭代法 + AMG 预处理（Iterative + AMG）—— 扩展之神
*   **应用场景**：自由度 $> 500$ 万，千万级/亿级超大规模计算，主要为三维实体正定/半正定结构问题。
*   **外挂核武**：
    *   **Hypre (BoomerAMG)**：目前工业界最成熟的代数多重网格库。
    *   **PETSc (KSP + PC)**：最顶级的 Krylov 子空间封装库（内置 CG, GMRES, BiCGSTAB，并可无缝调用 Hypre 作为 PC）。
*   **UFC 调用策略**：L5_RT 只管调 `L2_NM_Solver_Proc_Solve`，底层通过 `_Brg.f90` 将矩阵指针移交给 PETSc。

## 4. 架构落地设计 (`L2_NM` 模块)

基于以上战略，`L2_NM` 层将不再有冗长的数学推导代码，而是变成一个轻量级的高速接口网关：

*   **`L2_NM_Solver_Def.f90`**：定义统一的求解器配置 `TYPE`。里面只需配置 `solver_type = 'MKL_PARDISO'` 或是 `solver_type = 'PETSC_CG_AMG'`。
*   **`L2_NM_Matrix_Brg.f90`**：矩阵桥接器。负责把 UFC 内部的 CSR 数组指针，按照 C-Interop 规范，零拷贝（Zero-Copy）转换为 PETSc 的 `Mat` 对象或 PARDISO 的数组指引。
*   **`L2_NM_Solver_Proc.f90`**：向 `L5_RT` 提供暴露极简的三步走接口：
    1.  `_Proc_GlbItrSymbolic`（符号分解）
    2.  `_Proc_GlbItrFactor`（数值分解/AMG构建）
    3.  `_Proc_GlbItrSolve`（回代求解）

把解方程的脏活和苦活全部扔给世界上最聪明的一批数学家（PETSc/MKL），UFC 集中全部火力，猛攻非线性、接触、材料、算法等工程物理难题！这就是超越商业软件的最优系统工程学！
