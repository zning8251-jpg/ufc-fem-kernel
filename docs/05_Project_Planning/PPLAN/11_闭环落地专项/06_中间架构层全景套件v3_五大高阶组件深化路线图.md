## 各组件核心设计思路与卡点约束

### 重点 1：AI Proxy (AI 代理与批处理执行件)
*   **零拷贝内存规约 (Zero-Copy Memory Protocol)**：详细规定 Fortran 中 `TYPE(XX_State)` 的大数组必须遵守 `BIND(C)` 规范。需要明确 Fortran 的列优先（Column-major）与 PyTorch/C++ 的行优先（Row-major）映射关系，确保内存指针零损耗直通。
*   **打包与解包机制 (Batched Pack/Unpack)**：突破传统 FEM 的单元级单点循环。规划统一的 Batch Buffer，将 $N$ 个高斯点的应变/历史变量集中打包（Pack），通过单次 FFI 调用交给 GPU 推理，随后批量解包（Unpack）写回状态。

### 重点 2：Extension (第三方沙盒屏障件 / UEL & UMAT)
*   **标准黑盒签名映射**：规划 UFC 标准的五参/六参（Desc/State/Algo/Ctx/Arg）如何“降维”适配 Abaqus 标准的扁平一维数组序列（如 `PROPS`, `DSTRAN`, `STATEV`）。
*   **NaN 检疫与异常熔断机制**：建立严格的沙盒进出关检查模板。第三方代码返回时，强制对刚度矩阵（DDSDDE）和应力（STRESS）执行 `IEEE_IS_NAN` 检测，一旦发现异常立即熔断并向上层返回标准 UFC Error Code，绝对阻断污染全局刚度矩阵。

### 重点 3：Bridge 与 Populate (防腐桥与数据灌入件)
*   **对象树扁平化协议 (Tree Flattening Protocol)**：L3 层的面向对象树状数据如何被“拍扁”。规定由谁分配大数组，以及如何将多态的材料参数展平成纯粹的连贯 `REAL*8` 内存块。
*   **热路径与冷数据隔离红线**：明确时间轴约束。Populate 只能在 `Init` 时相发生；求解期的循环热路径只能走 Bridge 切片，绝对禁止在热路径中读取 L3 的树状结构或执行动态分配（`ALLOCATE`）。

### 重点 4：Iterator (高级迭代与路径控制件)
*   **状态双缓冲与回滚机制 (Double-buffering & Rollback)**：针对弧长法（ARC）、L-BFGS 的试错需求，规划 `_State` 的版本控制机制（保存 $t$ 与 $t+\Delta t$ 状态）。迭代器拥有发出 `Revert` 或 `Commit` 指令的最高权限。
*   **控制权反转回调 (IoC / Callback)**：通过 Fortran 标准 `PROCEDURE POINTER` 建立回调总线。主线性求解器与非线性策略完全解耦，迭代器可通过回调动态缩放载荷乘子或切换算法。

### 重点 5：Parallel Comm (并行通信与幽灵同步件)
*   **时间轴卡点 (Sync Timeline)**：严格定义 MPI 交换发生在残差组装（Assemble）之前还是之后。
*   **Ghost Nodes 映射表与规约**：制定跨节点数据结构，管理“本核拥有节点（Owned）”与“相邻核幽灵节点（Ghost）”。规范残差力的 `MPI_REDUCE` 和节点位移的 `MPI_SCATTER` 标准动作模式。
